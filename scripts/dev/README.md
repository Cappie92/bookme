# scripts/dev

**Dev-only** и **one-off** скрипты: reseed, диагностика, локальные e2e-обвязки, отладочные утилиты.

- **production / ops** остаётся в `scripts/prod/`.
- Перенос скриптов из корня `scripts/` сюда — **поэтапно** (см. `docs/REPO_AND_DEPLOY_HYGIENE.md`), без ломки путей в CI.

Текущие скрипты в родительском `scripts/` не удаляем в этом pass — только договорённость о структуре.
