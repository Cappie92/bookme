# ADR-0005: Управление временными слотами (10-минутная гранулярность)

**Дата:** 2024-10-21

**Статус:** Принято

**Контекст:** Команда разработки DeDato

---

## Контекст и проблема

Система бронирования должна управлять доступными временными слотами для записи к мастерам. Необходимо решить:

- Какая гранулярность времени бронирования?
- Как генерировать доступные слоты?
- Как проверять конфликты?
- Как освобождать слоты при отмене?
- Как учитывать длительность услуг?

### Требования

**Функциональные:**
- Услуги различной длительности (от 10 минут до 8 часов)
- Проверка доступности слотов
- Предотвращение двойного бронирования
- Освобождение слотов при отмене
- Гибкость для различных типов услуг

**Нефункциональные:**
- Быстрая генерация слотов (<100ms)
- Точность до минуты
- Удобный UX (не слишком много опций, но и достаточная гибкость)

**Ограничения:**
- Мастера работают в фиксированные часы (например, 8:00-20:00)
- Нужна совместимость с существующими данными (было 30 минут)

## Рассмотренные варианты

### Вариант 1: 30-минутная гранулярность

**Описание:** Слоты кратны 30 минутам (8:00, 8:30, 9:00...)

**Плюсы:**
- Простота реализации
- Меньше вариантов = быстрее выбор
- Стандартный подход в индустрии

**Минусы:**
- Негибко для коротких услуг (15 минут маникюр)
- Потеря времени при округлении
- Сложнее оптимизировать расписание

### Вариант 2: 10-минутная гранулярность

**Описание:** Слоты кратны 10 минутам (8:00, 8:10, 8:20...)

**Плюсы:**
- Гибкость для различных услуг (10, 20, 30, 40, 50 минут)
- Более эффективное использование времени мастера
- Возможность точной оптимизации расписания
- Покрывает большинство реальных длительностей услуг

**Минусы:**
- Больше вариантов в UI (может быть overwhelming)
- Сложнее визуализация в календаре
- Больше проверок конфликтов

### Вариант 3: 5-минутная гранулярность

**Описание:** Слоты кратны 5 минутам

**Плюсы:**
- Максимальная гибкость

**Минусы:**
- Слишком детально для салонов красоты
- Сложность для пользователя
- Оверкилл для текущих требований

### Вариант 4: Свободный выбор времени (любая минута)

**Описание:** Можно выбрать любое время (например, 14:23)

**Плюсы:**
- Максимальная гибкость

**Минусы:**
- Хаос в расписании
- Сложная визуализация
- Ошибки пользователей
- Плохой UX

## Принятое решение

**Выбран:** Вариант 2 (10-минутная гранулярность)

### Обоснование

10-минутная гранулярность обеспечивает оптимальный баланс между гибкостью и простотой:

1. **Гибкость:**
   - 10 мин: Консультация, экспресс-услуги
   - 20 мин: Быстрые процедуры
   - 30 мин: Стандартные услуги (маникюр)
   - 60 мин: Массаж, комплексный уход
   - 120 мин: Окрашивание волос

2. **Эффективность:**
   - Минимизация простоя между записями
   - Возможность плотного заполнения расписания
   - Точная оптимизация времени

3. **UX:**
   - Не overwhelming (6 вариантов на час vs 60 при минутной точности)
   - Легко визуализировать в календаре (высота ячейки пропорциональна времени)
   - Интуитивно понятно

4. **Реальность:**
   - Большинство услуг салонов красоты кратны 10-30 минутам
   - Соответствует рабочим практикам

## Последствия

### Положительные

- Гибкое управление расписанием
- Эффективное использование времени мастера
- Возможность точной оптимизации
- Покрытие 95% реальных сценариев
- Хороший баланс гибкости и простоты

### Отрицательные

- Более сложная логика по сравнению с 30-минутными слотами
- Больше данных для обработки (в 3 раза больше слотов)
- Необходимость валидации кратности 10 минутам

### Риски

**Риск 1:** Производительность при генерации большого количества слотов

**Митигация:**
- Кэширование доступных слотов на день
- Генерация слотов только для запрошенного периода
- Оптимизированные SQL запросы

**Риск 2:** UX может быть overwhelming для пользователей

**Митигация:**
- Группировка слотов по часам
- Скрытие недоступных слотов
- Умные рекомендации (популярное время)

## Детали реализации

### Генерация слотов

```python
# backend/services/scheduling.py
def generate_time_slots(start_time: datetime, end_time: datetime, slot_minutes: int = 10) -> List[datetime]:
    """
    Генерирует временные слоты с заданной гранулярностью.
    
    Args:
        start_time: Начало рабочего дня
        end_time: Конец рабочего дня
        slot_minutes: Гранулярность в минутах (по умолчанию 10)
    
    Returns:
        Список временных меток слотов
    """
    slots = []
    current = start_time
    
    while current < end_time:
        slots.append(current)
        current += timedelta(minutes=slot_minutes)
    
    return slots

# Пример: 8:00 - 12:00
# Результат: [8:00, 8:10, 8:20, 8:30, ..., 11:50]
```

### Валидация времени

```python
def validate_booking_time(start_time: datetime, duration_minutes: int) -> bool:
    """
    Проверяет, что время записи кратно 10 минутам.
    """
    if start_time.minute % 10 != 0:
        raise ValueError("Время начала должно быть кратно 10 минутам")
    
    if duration_minutes % 10 != 0:
        raise ValueError("Длительность должна быть кратна 10 минутам")
    
    return True

# Примеры:
# validate_booking_time(datetime(2024, 10, 21, 14, 0), 30)   # ✓ OK
# validate_booking_time(datetime(2024, 10, 21, 14, 10), 60)  # ✓ OK
# validate_booking_time(datetime(2024, 10, 21, 14, 15), 30)  # ✗ Error: не кратно 10
```

### Проверка конфликтов

```python
def check_slot_availability(
    master_id: int,
    start_time: datetime,
    duration_minutes: int,
    db: Session
) -> bool:
    """
    Проверяет доступность временного слота.
    """
    end_time = start_time + timedelta(minutes=duration_minutes)
    
    # Ищем пересечения с существующими бронированиями
    conflicts = db.query(Booking).filter(
        Booking.master_id == master_id,
        Booking.status.in_([BookingStatus.CREATED, BookingStatus.AWAITING_CONFIRMATION]),
        Booking.start_time < end_time,
        Booking.end_time > start_time
    ).count()
    
    return conflicts == 0
```

### Расчет необходимых слотов

```python
def calculate_required_slots(duration_minutes: int, slot_size: int = 10) -> int:
    """
    Вычисляет количество 10-минутных слотов для услуги.
    
    Args:
        duration_minutes: Длительность услуги в минутах
        slot_size: Размер слота (по умолчанию 10)
    
    Returns:
        Количество слотов
    """
    return duration_minutes // slot_size

# Примеры:
# calculate_required_slots(30)  # 3 слота
# calculate_required_slots(60)  # 6 слотов
# calculate_required_slots(90)  # 9 слотов
```

### Доступные слоты для услуги

```python
def get_available_slots_for_service(
    master_id: int,
    service_duration: int,
    date: datetime.date,
    db: Session
) -> List[datetime]:
    """
    Возвращает список доступных временных слотов для услуги заданной длительности.
    """
    # Получаем рабочее расписание мастера
    schedule = db.query(MasterSchedule).filter(
        MasterSchedule.master_id == master_id,
        MasterSchedule.date == date
    ).first()
    
    if not schedule:
        return []
    
    # Генерируем все возможные слоты
    all_slots = generate_time_slots(
        datetime.combine(date, schedule.start_time),
        datetime.combine(date, schedule.end_time)
    )
    
    # Фильтруем доступные слоты
    available = []
    for slot in all_slots:
        if check_slot_availability(master_id, slot, service_duration, db):
            available.append(slot)
    
    return available
```

### Освобождение слотов при отмене

```python
def free_booking_slots(booking: Booking, current_time: datetime) -> List[datetime]:
    """
    Освобождает временные слоты при отмене записи.
    
    Логика:
    - Если отмена ДО start_time: освобождаются ВСЕ слоты
    - Если отмена ПОСЛЕ start_time: только будущие 10-мин слоты
    """
    freed_slots = []
    
    if current_time < booking.start_time:
        # Отмена до начала - освобождаем все
        slot = booking.start_time
        while slot < booking.end_time:
            freed_slots.append(slot)
            slot += timedelta(minutes=10)
    else:
        # Отмена после начала - только будущие слоты
        # Округляем текущее время вверх до ближайших 10 минут
        next_slot_minute = ((current_time.minute // 10) + 1) * 10
        if next_slot_minute >= 60:
            next_slot = current_time.replace(hour=current_time.hour + 1, minute=0)
        else:
            next_slot = current_time.replace(minute=next_slot_minute)
        
        while next_slot < booking.end_time:
            freed_slots.append(next_slot)
            next_slot += timedelta(minutes=10)
    
    return freed_slots

# Пример:
# Запись: 14:00 - 15:00
# Текущее время: 14:25
# Освобождаются: 14:30, 14:40, 14:50
```

## Примечания

**Типичные длительности услуг:**
- 10 мин: Консультация, экспресс-процедуры
- 20-30 мин: Маникюр, педикюр
- 40-60 мин: Массаж, стрижка
- 90-120 мин: Окрашивание волос, комплексный уход
- 180+ мин: Сложные процедуры, комбинированные услуги

**Производительность:**
- Генерация слотов на день (12 часов): ~72 слота
- Запрос проверки конфликтов: <10ms
- Кэширование доступных слотов: 5 минут TTL
