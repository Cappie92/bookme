# ClientDashboard — Точечный micro-fix (3 проблемы)

## Дата: 2026-02-05

## Контекст
После предыдущего патча обнаружены 3 критические проблемы:
1. Сердечки (favorites) не синхронизируются между секциями
2. Отступы/выравнивание иконок в "Действия" неравномерные
3. "Избранные" отображаются как список строк вместо карточек

## Исправленные проблемы

### 1. FIX FAVORITES SYNC (сердечки не закрашиваются синхронно)

**ПРИЧИНА**: 
FavoriteButton в таблицах "Будущие записи" и "Прошедшие записи" НЕ получал prop `onFavoriteChange={handleFavoriteChange}`. Из-за этого:
- Клик по сердечку обновлял только внутренний контекст `FavoritesContext` через `useFavorites`
- Локальный state `favoriteMasterIds` в `ClientDashboard` НЕ обновлялся
- Другие секции не знали об изменении и не перерисовывались

**РЕШЕНИЕ**:
Добавил `onFavoriteChange={handleFavoriteChange}` во ВСЕ `FavoriteButton` компоненты:
- Основная таблица "Будущие записи" (2 места: `indie_master` и `master`)
- Основная таблица "Прошедшие записи" (2 места)
- Модальное окно "Все будущие записи" (2 места)
- Модальное окно "Все прошедшие записи" (2 места)
- В секции "Избранные" уже был callback (не трогал)

**Как работает**:
1. Клик по сердечку → FavoriteButton вызывает API
2. После успешного ответа → FavoriteButton вызывает `onFavoriteChange(type, itemId, isFavorite)`
3. `handleFavoriteChange` обновляет `favoriteMasterIds` (новый Set с нормализованным `Number(itemId)`)
4. React перерисовывает все секции, которые зависят от `favoriteMasterIds`
5. Все сердечки обновляются мгновенно

**Файлы**: `frontend/src/pages/ClientDashboard.jsx`
- Строки ~1108-1120 (Будущие, основная таблица)
- Строки ~1240-1252 (Прошедшие, основная таблица)
- Строки ~1846-1858 (Будущие, модалка)
- Строки ~1974-1986 (Прошедшие, модалка)

### 2. FIX ACTIONS SPACING (отступы/выравнивание иконок "Действия")

**ПРИЧИНА**:
- Разные размеры кликабельных зон: где-то `p-2`, где-то без padding
- Разные gap: `gap-3` vs `gap-2`
- Иконки без единой обёртки фиксированного размера
- Ячейка "Действия" выравнивалась по-разному: `justify-start` vs без выравнивания

**РЕШЕНИЕ**:
Унифицировал все action buttons:
1. **Единая обёртка для каждой иконки**: `<div className="inline-flex items-center justify-center w-8 h-8">`
2. **Единый gap**: `gap-2` везде
3. **Единый hover**: `hover:bg-{color}-50 rounded-md p-1`
4. **Единое выравнивание ячейки**: `<td className="py-2 px-3 text-right">` + `<div className="flex items-center justify-end gap-2">`

Применил изменения:
- Основная таблица "Будущие записи": сердечко + редактировать + отменить
- Основная таблица "Прошедшие записи": сердечко + повторить + заметка + dislike (уже было правильно)
- Модальное окно "Все будущие записи": сердечко + редактировать + отменить
- Модальное окно "Все прошедшие записи": уже было правильно

**Файлы**: `frontend/src/pages/ClientDashboard.jsx`
- Строки ~1103-1149 (Будущие, основная таблица)
- Строки ~1841-1883 (Будущие, модалка)

### 3. FAVORITES UI: вернуть карточки, но компактные

**ПРИЧИНА**:
В предыдущем патче секция "Избранные" была переделана в список строк (`space-y-2`), что не соответствовало дизайну.

**РЕШЕНИЕ**:
Вернул карточки в grid layout, но сделал их компактными:

**Layout секции**:
- `grid grid-cols-2 gap-3` (2 колонки на десктопе)
- Вместо `space-y-2` (вертикальный список)

**Карточка** (`renderFavoriteCard`):
- `bg-white border border-gray-200 rounded-lg shadow-sm p-3`
- `hover:shadow-md transition-shadow` для интерактивности
- Внутри: `flex items-start justify-between gap-2`
  - Слева: тип ("Мастер") + имя (кликабельное зелёное)
  - Справа: сердечко (маленькое, для снятия из избранного)
- Компактные отступы: `p-3` вместо `p-4`, `mb-1` для подписи
- `truncate` для длинных имён

**Пустое состояние**: `text-sm text-gray-500` без лишних отступов

**Файлы**: `frontend/src/pages/ClientDashboard.jsx`
- Строки ~1333-1339 (grid layout)
- Строки ~432-469 (renderFavoriteCard)
