# Исправление ошибки 500 в /api/master/dashboard/stats

## Проблема
Эндпоинт возвращает 500 ошибку при загрузке статистики дашборда мастера.

## Принятые меры
1. Добавлено логирование на каждом шаге выполнения функции
2. Добавлена проверка на None для `current_period` перед использованием
3. Добавлена безопасная сериализация дат и времени
4. Улучшена обработка ошибок с детальным логированием

## Файл для обновления
`backend/routers/master.py`

## Команда для обновления на сервере
```bash
cd /Users/s.devyatov/DeDato
scp backend/routers/master.py root@193.160.208.206:/home/root/dedato/backend/routers/master.py
ssh root@193.160.208.206 'cd /home/root/dedato && docker-compose -f docker-compose.prod.yml restart backend'
```

## Что проверить после обновления
1. В логах должны появиться строки "🔍 Шаг 1", "🔍 Шаг 2" и т.д.
2. Последняя строка перед ошибкой покажет, на каком шаге происходит сбой
3. В логах должен появиться полный traceback ошибки

## Возможные причины ошибки
1. Проблема с сериализацией объектов date/datetime в JSON
2. Ошибка при обращении к полям словаря `current_period` или `period_data`
3. Ошибка в SQL-запросах к базе данных
