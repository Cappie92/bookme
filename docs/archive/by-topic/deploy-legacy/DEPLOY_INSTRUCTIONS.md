# Инструкции по деплою DeDato

## 📦 Архив создан

Архив проекта создан: `dedato_deploy_YYYYMMDD_HHMMSS.tar.gz`

## 🚀 Шаги для деплоя на сервер

### Вариант 1: Автоматический деплой (если SSH настроен)

```bash
# 1. Загрузите архив на сервер
scp dedato_deploy_*.tar.gz root@193.160.208.206:/home/root/

# 2. Подключитесь к серверу
ssh root@193.160.208.206

# 3. На сервере выполните:
cd /home/root
rm -rf dedato_old
mv dedato dedato_old 2>/dev/null || true
mkdir -p dedato
cd dedato
tar -xzf ../dedato_deploy_*.tar.gz
rm ../dedato_deploy_*.tar.gz

# 4. Остановите старые контейнеры и запустите новые
docker-compose -f docker-compose.prod.yml down 2>/dev/null || true
docker-compose -f docker-compose.prod.yml up -d --build

# 5. Проверьте статус
docker-compose -f docker-compose.prod.yml ps
docker-compose -f docker-compose.prod.yml logs backend --tail=50
docker-compose -f docker-compose.prod.yml logs frontend --tail=50
```

### Вариант 2: Ручной деплой через веб-интерфейс или FTP

1. Загрузите архив `dedato_deploy_*.tar.gz` на сервер любым способом
2. Подключитесь к серверу по SSH
3. Выполните команды из варианта 1 (шаги 3-5)

## ✅ Проверка после деплоя

1. **Проверьте API:**
   ```bash
   curl http://193.160.208.206:8000/health
   ```
   Должен вернуть: `{"status":"healthy","service":"DeDato API"}`

2. **Проверьте фронтенд:**
   ```bash
   curl -I http://193.160.208.206:5173
   ```
   Должен вернуть: `200 OK`

3. **Проверьте в браузере:**
   - Откройте http://193.160.208.206:5173
   - Проверьте, что страница загружается
   - Проверьте функциональность избранного
