# Увеличение ширины карточек в сетке «Быстрые скидки»

## Проблема

На узких экранах (2 колонки) карточки были слишком узкими, из-за чего в edit-режиме кнопки «Сохранить» и «Отмена» обрезались («Сохр…», «Отме…») и карточка визуально увеличивалась по высоте.

## Решение

Уменьшен боковой отступ контейнера (`paddingHorizontal`) с 6px до 2px, что увеличило доступную ширину для карточек на 8px (4px с каждой стороны). Это автоматически увеличило `listWidth` (измеряется через `onLayout`) и пересчитало `ITEM_WIDTH`, сделав карточки шире.

---

## Unified diff

### Стили — уменьшение paddingHorizontal контейнера

```diff
const styles = StyleSheet.create({
  container: {
-   paddingHorizontal: 6,
+   paddingHorizontal: 2,
    paddingVertical: 16,
  },
```

---

## Что изменилось

### A) Найдены текущие значения

- **H_PADDING (paddingHorizontal контейнера)**: было `6`, стало `2`
- **COL_GAP**: `8` (не изменялся)
- **ROW_GAP**: `12` (не изменялся)
- **ITEM_WIDTH**: вычисляется автоматически как `Math.floor((listWidth - COL_GAP) / 2)`, где `listWidth` измеряется через `onLayout` контейнера `templatesContainer`
- **wrapperStyle**: динамически задается с `marginRight: isLeft ? COL_GAP / 2 : 0` и `marginLeft: !isLeft ? COL_GAP / 2 : 0` (не изменялся)

### B) Уменьшен боковой отступ контейнера

- `container.paddingHorizontal`: `6` → `2` (уменьшение на 4px с каждой стороны)
- Это увеличило `listWidth` на 8px (4px × 2)
- `ITEM_WIDTH` автоматически пересчитался и увеличился на 4px на карточку
- `COL_GAP` остался неизменным (`8`)

### C) Проверка вертикальных отступов

- `editInputRow.marginBottom`: `6` (единственный вертикальный отступ между input и кнопками) — оставлен без изменений
- Высота карточки в обычном режиме не изменилась
- `templateCardInner.paddingBottom`: `12` — не изменялся

### D) Edit-режим уже настроен корректно

Кнопки уже имеют правильные стили (из предыдущих фиксов):
- `saveButton` / `cancelButton`: `flex: 1`, `minWidth: 0`, `paddingHorizontal: 10`, `height: 36`
- `editActionsRow`: `width: '100%'`, `minWidth: 0`
- Текст кнопок: `numberOfLines={1}`, `ellipsizeMode="tail"`
- Отступ между кнопками: `marginRight: 8` у `saveButton`

С увеличенной шириной карточки кнопки теперь должны помещаться без обрезки текста.

### E) Механика «левая влево / правая вправо» сохранена

- Левая карточка: `marginRight: COL_GAP / 2` (4px), `marginLeft: 0`
- Правая карточка: `marginLeft: COL_GAP / 2` (4px), `marginRight: 0`
- Карточки по-прежнему «разъезжаются» к краям экрана

---

## Результат

### Увеличение ширины карточек

- **Было**: `paddingHorizontal: 6` → карточки узкие
- **Стало**: `paddingHorizontal: 2` → карточки шире на 8px (4px с каждой стороны)
- **COL_GAP**: остался `8` (не уменьшался)

### Автоматический пересчет

- `listWidth` увеличился на 8px (из-за уменьшения padding контейнера)
- `ITEM_WIDTH` автоматически пересчитался: `(listWidth - COL_GAP) / 2`
- Каждая карточка стала шире на 4px

### Edit-режим

- Кнопки «Сохранить» и «Отмена» теперь помещаются по ширине
- Текст не обрезается благодаря увеличенной ширине карточки
- Карточка не раздувается по высоте (вертикальные отступы не менялись)
