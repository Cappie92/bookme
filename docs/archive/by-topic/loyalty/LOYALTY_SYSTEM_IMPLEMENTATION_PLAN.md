# План реализации системы лояльности для мастеров

## Обзор
Система лояльности позволяет мастерам начислять баллы клиентам за посещения и позволяет клиентам частично или полностью оплачивать услуги баллами.

## 1. База данных

### 1.1. Создание миграции Alembic

**Файл:** `backend/alembic/versions/XXXX_add_loyalty_points_system.py`

#### Таблица `loyalty_settings`
Настройки программы лояльности для каждого мастера:
- `id` (Integer, PK)
- `master_id` (Integer, FK -> masters.id, UNIQUE)
- `is_enabled` (Boolean, default=False) - включена ли программа лояльности
- `accrual_percent` (Integer, nullable=True) - процент начисления (1-100)
- `max_payment_percent` (Integer, nullable=True) - максимальный % оплаты баллами (1-100)
- `points_lifetime_days` (Integer, nullable=True) - срок жизни баллов в днях (NULL = бесконечно)
- `created_at` (DateTime)
- `updated_at` (DateTime)

Индексы:
- `idx_loyalty_settings_master` на `master_id`

#### Таблица `loyalty_transactions`
Транзакции начисления и списания баллов:
- `id` (Integer, PK)
- `master_id` (Integer, FK -> masters.id)
- `client_id` (Integer, FK -> users.id)
- `booking_id` (Integer, FK -> bookings.id, nullable=True) - ID записи, за которую начислены баллы
- `transaction_type` (String) - 'earned' (начисление) или 'spent' (списание)
- `points` (Integer) - количество баллов (положительное число)
- `earned_at` (DateTime) - дата начисления (для earned) или списания (для spent)
- `expires_at` (DateTime, nullable=True) - дата истечения (только для earned, NULL = бесконечно)
- `service_id` (Integer, FK -> services.id, nullable=True) - ID услуги, за которую начислены баллы
- `created_at` (DateTime)

Индексы:
- `idx_loyalty_transactions_master_client` на `master_id, client_id`
- `idx_loyalty_transactions_client` на `client_id`
- `idx_loyalty_transactions_booking` на `booking_id`
- `idx_loyalty_transactions_expires` на `expires_at` (WHERE expires_at IS NOT NULL)
- `idx_loyalty_transactions_type` на `transaction_type`

#### Добавление поля в таблицу `bookings`
- `loyalty_points_used` (Integer, nullable=True, default=0) - количество списанных баллов

### 1.2. Создание моделей SQLAlchemy

**Файл:** `backend/models.py`

Добавить модели:
- `LoyaltySettings` - настройки лояльности мастера

Связи:
- `LoyaltySettings.master` -> `Master`
- `LoyaltyTransaction.master` -> `Master`

## 2. Backend API

### 2.1. Роутер лояльности

**Файл:** `backend/routers/loyalty.py` (создать новый или дополнить существующий)

#### Эндпоинты для мастера:

**GET `/api/master/loyalty/settings`**
- Получить настройки лояльности мастера
- Проверка доступа: `has_finance_access`
- Возвращает настройки или создает дефолтные (если нет)

**PUT `/api/master/loyalty/settings`**
- Обновить настройки лояльности
- Проверка доступа: `has_finance_access`
- Валидация: проценты от 1 до 100 (целые числа)
- Параметры:
  - `is_enabled` (Boolean)
  - `accrual_percent` (Integer, 1-100, optional)
  - `max_payment_percent` (Integer, 1-100, optional)
  - `points_lifetime_days` (Integer, nullable, optional) - 14, 30, 60, 90, 180, 365, или NULL

**GET `/api/master/loyalty/history`**
- Получить историю операций по баллам
- Проверка доступа: `has_finance_access`
- Фильтры: `client_id` (optional), `transaction_type` (optional), `start_date`, `end_date`
- Пагинация
- Возвращает список транзакций с информацией о клиентах и услугах

**GET `/api/master/loyalty/stats`**
- Получить статистику по баллам
- Проверка доступа: `has_finance_access`
- Возвращает:
  - Общее количество выданных баллов
  - Общее количество списанных баллов
  - Текущий баланс всех клиентов (начислено - списано)
  - Количество активных клиентов с баллами

#### Эндпоинты для клиента:

**GET `/api/client/loyalty/points`**
- Получить все баллы клиента у всех мастеров
- Группировка по мастерам
- Возвращает для каждого мастера:
  - ID мастера
  - Имя мастера
  - Текущий баланс (сумма активных баллов)
  - Детали по транзакциям (если нужно)

**GET `/api/client/loyalty/points/{master_id}`**
- Получить баллы клиента у конкретного мастера
- Возвращает баланс и историю транзакций

**GET `/api/client/loyalty/available/{master_id}`**
- Получить доступные к использованию баллы у мастера (FIFO, без истекших)
- Используется при отображении в форме записи
- Возвращает:
  - `available_points` - количество доступных баллов
  - `max_spendable` - максимальная сумма, которую можно списать (с учетом лимита мастера)

### 2.2. Интеграция с подтверждением записи

**Файл:** `backend/routers/accounting.py`

Модифицировать функцию `confirm_booking`:

1. После создания `BookingConfirmation` и `Income`:
   - Проверить, есть ли у мастера включенная программа лояльности
   - Если да:
     - Получить `payment_amount` из booking
     - Вычесть `loyalty_points_used` (если есть)
     - Вычислить сумму для начисления: `payment_amount - loyalty_points_used`
     - Получить настройки лояльности мастера
     - Вычислить баллы: `(payment_amount - loyalty_points_used) * accrual_percent / 100`
     - Вычислить дату истечения: `earned_at + points_lifetime_days` (или NULL)
     - Создать транзакцию типа 'earned'

2. Если были списаны баллы:
   - Уменьшить `confirmed_income` и `master_earnings` в `Income` на сумму списанных баллов
   - Это уже учтено в `payment_amount - loyalty_points_used`, но нужно убедиться, что `Income` создается правильно

### 2.3. Резервирование и списание баллов при создании записи

**Файл:** `backend/routers/bookings.py` или `backend/routers/client.py`

Модифицировать создание бронирования:

1. В эндпоинте создания записи (`POST /client/bookings` или `/bookings/public`):
   - Принять параметр `use_loyalty_points` (Boolean) из запроса
   - Если `use_loyalty_points = True`:
     - Получить доступные баллы клиента у мастера (FIFO)
     - Получить настройки лояльности мастера
     - Вычислить максимальную сумму списания: `service_price * max_payment_percent / 100`
     - Вычислить сумму списания: `min(available_points, max_spendable, service_price)`
     - Записать `loyalty_points_used = сумма списания` в booking
     - НЕ создавать транзакции списания здесь (только при подтверждении)

2. При отмене записи:
   - Если запись имела `loyalty_points_used > 0`, обнулить это поле (баллы вернутся, так как не были списаны)

3. При подтверждении записи:
   - Если `booking.loyalty_points_used > 0`:
     - Списать баллы по FIFO (старые первыми)
     - Создать транзакции типа 'spent'
     - Уменьшить доход в `Income` на сумму списанных баллов

### 2.4. Утилиты для работы с баллами

**Файл:** `backend/utils/loyalty.py` (создать)

Функции:
- `get_available_points(db, master_id, client_id)` - получить доступные баллы (FIFO)
- `calculate_points_to_spend(available_points, service_price, max_payment_percent)` - вычислить сумму списания
- `spend_points(db, master_id, client_id, amount, booking_id)` - списать баллы по FIFO
- `earn_points(db, master_id, client_id, amount, booking_id, service_id, lifetime_days)` - начислить баллы
- `get_client_balance_by_master(db, client_id, master_id)` - баланс клиента у мастера
- `cleanup_expired_points(db)` - удалить/пометить истекшие баллы (для периодической задачи)

## 7. Важные моменты реализации

### 7.1. Алгоритм списания баллов (FIFO)
1. Получить все транзакции типа 'earned' для клиента у мастера, отсортированные по `earned_at` (ASC)
2. Исключить истекшие (где `expires_at < now()`)
3. Списывать по порядку (старые первыми)
4. Если одна транзакция частично списывается, создавать новую транзакцию 'spent' на списанную сумму
5. При полном списании создавать транзакцию 'spent' на полную сумму транзакции 'earned'

### 7.2. Расчет начисления баллов
- Формула: `(payment_amount - loyalty_points_used) * accrual_percent / 100`
- Округление: до целого числа баллов (round)
- Дата истечения: `earned_at + points_lifetime_days` (или NULL если бесконечно)

### 7.3. Расчет списания баллов
- Максимальная сумма списания: `min(service_price * max_payment_percent / 100, available_points, service_price)`
- Доступные баллы вычисляются по FIFO (без учета истекших)

### 7.4. Валидация настроек
- Все проценты должны быть целыми числами от 1 до 100
- `points_lifetime_days` может быть NULL (бесконечно) или одним из значений: 14, 30, 60, 90, 180, 365

### 7.5. Проверка доступа
- Функция лояльности доступна только мастерам с подпиской Pro/Premium (`has_finance_access = True`)
- Все эндпоинты мастера должны проверять этот доступ
- В UI неактивная кнопка должна показывать подсказку о необходимости подписки

## 8. Тестовые сценарии

1. **Создание настроек лояльности**
   - Мастер создает настройки
   - Валидация процентов
   - Сохранение настроек

2. **Начисление баллов**
   - Мастер подтверждает запись
   - Баллы начисляются с правильным процентом
   - Дата истечения устанавливается корректно

3. **Списание баллов**
   - Клиент выбирает "Потратить" при записи
   - Баллы резервируются
   - При подтверждении баллы списываются по FIFO
   - Доход мастера уменьшается на сумму списанных баллов

4. **Истечение баллов**
   - Баллы с истекшим сроком не показываются клиенту
   - Баллы с истекшим сроком не участвуют в списании

5. **Отмена записи**
   - При отмене резервирование снимается
   - Баллы остаются доступными

6. **Проверка доступа**
   - Мастер без подписки Pro не может использовать функцию
   - Кнопка неактивна с подсказкой
