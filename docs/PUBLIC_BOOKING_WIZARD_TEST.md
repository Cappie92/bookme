# Проверка публичного визарда записи (Mobile + Web)

## Mobile

### Запуск
- Установить приложение: `cd mobile && npx expo run:ios`
- Deep link: `xcrun simctl openurl booted "dedato://m/m-TK5E3n9R"`

### Сценарии

1. **Полный флоу без логина**
   - Открыть по deep link (или зайти на /m/m-TK5E3n9R без токена).
   - Выбрать услугу → выбрать дату → выбрать время.
   - Нажать «Записаться» → должен открыться /login.
   - Войти → авто-возврат на /m/[slug] → создание брони из draft → экран успеха.
   - На экране успеха: «Добавить в календарь», «Перейти в мои записи» (если авторизован), «Вернуться к записи».

2. **Состояния визарда**
   - Загрузка профиля: спиннер + скелет-карточка.
   - Пустые услуги: текст «У мастера пока нет услуг».
   - После выбора услуги: шаг «Дата» активен; если слотов нет — «Нет свободных дат на ближайшие 14 дней».
   - После выбора даты: шаг «Время»; если слотов на дату нет — «Нет свободного времени на выбранную дату».
   - Сводка перед CTA: Услуга / Дата / Время.

3. **booking_blocked**
   - Мастер с блокировкой записи (или мок): CTA «Запись недоступна», disabled.

4. **Safe area**
   - Контент не уходит под статус-бар и «back to Safari».
   - Модалки выбора услуги/даты открываются полностью.

---

## Web

### Запуск
- Фронт: `cd frontend && npm run dev`
- URL: `http://localhost:5173/m/m-TK5E3n9R`

### Сценарии

1. **Те же шаги**: услуга → дата → время → CTA. Без логина: draft → открытие модалки входа → после логина создание брони и success.
2. **Мобильный Safari**: шапка и smart banner не перекрывают «Выберите услугу» и кнопки.
3. **Пустые услуги**: «У мастера пока нет услуг».
4. **Success**: «Добавить в календарь», «Перейти в мои записи» только при авторизации.
