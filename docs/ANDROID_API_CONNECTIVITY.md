# Android: доступность API (backend) с эмулятора

## Как проверить backend на хосте

На машине, где запущен backend:

```bash
curl -i http://localhost:8000/health
```

Ожидаемый ответ: `HTTP/1.1 200 OK` и JSON вида `{"status":"healthy","service":"DeDato API"}`.

Запуск backend (из корня репо):

```bash
cd backend && python3 -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

- **`--host 0.0.0.0`** — сервер слушает на всех интерфейсах. Так эмулятор Android может достучаться до хоста через `adb reverse` (localhost на эмуляторе мапится на хост) или через `10.0.2.2:8000` (специальный адрес эмулятора = хост-машина).
- Если указать только `--host 127.0.0.1`, с эмулятора до backend по `10.0.2.2` можно не попасть на части конфигураций; для разработки предпочтительно `0.0.0.0`.

## Эндпоинт /health

- В **backend/main.py**: `@app.get("/health")` → `{"status": "healthy", "service": "DeDato API"}`.
- Быстрый ответ без БД, подходит для проверки доступности API.

## adb reverse и 10.0.2.2

- **adb reverse tcp:8000 tcp:8000** — порт 8000 на эмуляторе перенаправляется на порт 8000 на хосте. После этого приложение на эмуляторе может использовать **`http://localhost:8000`** или **`http://127.0.0.1:8000`** — запросы уйдут на хост.
- **10.0.2.2** — специальный IP в сети эмулятора Android, указывающий на хост. То есть **`http://10.0.2.2:8000`** с эмулятора = backend на хосте:8000.
- Если **reverse не включён**, то `localhost:8000` с эмулятора = порт 8000 **на самом эмуляторе**, а не на хосте → подключения к вашему uvicorn не будет.
- Итог: либо включаем `adb reverse tcp:8000 tcp:8000` и в приложении оставляем `API_URL=http://localhost:8000`, либо в приложении задаём `API_URL=http://10.0.2.2:8000` (без reverse).

## Проверка с эмулятора без curl/wget

На эмуляторе часто нет `curl`/`wget`. Варианты:

1. **Проверка порта (toybox nc):**
   ```bash
   adb shell toybox nc -z 127.0.0.1 8000
   ```
   Код выхода 0 — порт открыт (при включённом reverse это хост:8000).

2. **Диагностика из приложения:** в dev при `DEBUG_HTTP=1` на экране public booking показывается блок с результатом запроса к `/health` и к getPublicMaster (см. docs/ANDROID_PUBLIC_BOOKING_API_DIAG_REPORT.md).
