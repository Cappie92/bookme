# AllBookingsModal UI/UX Refactor — Diff-Summary

## Сделанные изменения

### 1. Frontend: `frontend/src/utils/dateFormat.js` (новый файл)
- `formatDateShort(value)` — дата в формате DD.MM.YY (год 2 знака)
- `formatTimeShort(value)` — время HH:MM

### 2. Backend: `backend/routers/master.py`
- В ответы `/api/master/bookings/future` и `/api/master/past-appointments` добавлены поля:
  - `client_master_alias` (string|null) — имя от мастера (alias)
  - `client_account_name` (string|null) — имя из аккаунта клиента (full_name)

### 3. Frontend: `frontend/src/components/AllBookingsModal.jsx`
- **Новая структура по умолчанию (без фильтров):**
  - Блок A: «Ближайшие (будущие)» — 3 записи, кнопка «Показать ещё» (+10)
  - Блок B: «Прошедшие — требуют подтверждения» — 3 записи, «Показать ещё» (+10)
  - Блок C: «Прошедшие — подтвержденные/отклоненные» — 10 записей, «Показать ещё» (+10)
- Фильтры вынесены под кнопку «Фильтры» (опциональный режим)
- Загрузка: future (50) + past (100) при открытии
- **Строка записи:** компактная; дата DD.MM.YY; услуга (bold); клиент — если `client_master_alias` есть → зелёный текст без телефона, иначе имя + телефон серым
- Убрано «Клиент», убрано двойное отображение телефона
- Используется `formatDateShort` / `formatTimeShort` из `dateFormat.js`

### 4. Frontend: `frontend/src/components/MasterDashboardStats.jsx`
- Замена форматирования даты на `formatDateShort` (DD.MM.YY) в блоках «Ближайшие» и «Прошедшие»
