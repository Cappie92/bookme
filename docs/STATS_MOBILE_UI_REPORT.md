# Экран «Статистика» (mobile) — отчёт по доработкам

**Дата:** 2026-01-29

## Изменённые файлы

| Файл | Изменения |
|------|-----------|
| `mobile/app/master/stats.tsx` | Единый контейнер `screenContent` с `paddingHorizontal: 16`, перенос блока «Период» под KPI, Pro-гейтинг через `isPro` + `useFocusEffect` refresh, __DEV__-лог |

## 1. Padding / отступы текста

**Было:** Тексты «За выбранный период...» и «Доступно в Pro» визуально прилипали к левому краю.

**Стало:**
- Введён единый контейнер `View` с `style={styles.screenContent}` и `paddingHorizontal: 16`.
- В нём размещён весь контент экрана (заголовок, ошибка, KPI, период, toggle, расширенная, топы, графики).
- `ScreenContainer` по‑прежнему даёт `padding: 16` для scroll content; обёртка добавляет ещё 16px по горизонтали. Итог: 32px слева/справа для всех секций, единообразно на iOS и Android.

**Стиль:**
```ts
screenContent: {
  paddingHorizontal: 16,
},
```

## 2. Pro-гейтинг «Расширенная статистика»

**Источник данных:**
- API: `GET /api/master/subscription/features` (сервис `getMasterFeatures`, `mobile/src/services/api/master.ts`).
- Хук: `useMasterFeatures` (`mobile/src/hooks/useMasterFeatures.ts`), кеш в AsyncStorage (TTL 15 мин), ключ `@master_features:${userId}`.
- Признак Pro: `has_extended_stats === true` в ответе. Бэкенд считает его по `service_functions` плана (id `2` = расширенная статистика). Поля `plan_name` / `plan_id` не используются для гейтинга.

**Почему было неверно:**
- После логина или апгрейда до Pro features могли браться из кеша (`has_extended_stats: false`).
- При переходе на экран «Статистика» повторный запрос не выполнялся, использовались старые features.
- В итоге вкладка «Расширенная» оставалась disabled, показывался «Доступно в Pro» даже для Pro.

**Что сделано:**
- Явно введено `isPro = features?.has_extended_stats === true`. В UI везде используются `isPro` и `hasExtendedStats` (= `isPro`).
- При фокусе экрана «Статистика» вызывается `refresh()` из `useMasterFeatures` через `useFocusEffect` (`@react-navigation/native`). Это сбрасывает кеш и заново запрашивает `/api/master/subscription/features`.
- Блок «Доступно в Pro» и ссылка «Управление подпиской» рендерятся только при `!isPro`.
- В __DEV__ добавлен лог в `useEffect` при изменении `features`: `has_extended_stats`, `plan_name`, `plan_id`, `isPro`. По нему можно проверить, что после перехода с подписок/логина приходят актуальные значения.

**Итог:** При Pro-аккаунте вкладка «Расширенная» доступна, «Доступно в Pro» не показывается. Контракт API не менялся.

## 3. Перенос выбора периода наверх

**Было:** Блок «Период» (день/неделя/месяц/квартал/год + prev/today/next) располагался в середине экрана, после топов по записям и доходу.

**Стало:** Блок «Период» перенесён сразу под KPI (вариант Б: под двумя KPI-карточками и подсказкой «За выбранный период...»).

**Почему вариант Б:**
- Сначала показываются заголовок и KPI за выбранный период — пользователь понимает, «что» смотрит.
- Сразу под ними — выбор периода, чтобы задать «за какой» период эти KPI и все остальные блоки.
- Ниже — переключатель Базовая/Расширенная, топы, графики. Порядок: контекст (период) → уточнение (базовая/расширенная) → детали (топы, графики). Остальные блоки не менялись, логика и данные те же.

## 4. Защиты и стили

- Единый `screenContent` с `paddingHorizontal: 16` для всех секций.
- `isPro` считается один раз и используется в условиях рендера и `disabledIndexes` сегмента.

## 5. Линт / типизация

- ESLint по `app/master/stats.tsx`: без ошибок (конфиг не覆盖ет файл — предупреждение о «no matching configuration»).
- `tsc --noEmit`: в проекте есть прежние ошибки в других файлах (finance, index и т.д.). В `stats.tsx` остаются известные замечания по `style`-массивам (`ViewStyle`), к нашим правкам не относящиеся.

## 6. Smoke checklist (проверка вручную)

1. **Отступы**
   - [ ] iOS: открыть «Статистика», убедиться, что «За выбранный период...», «Доступно в Pro» (если есть), заголовки секций не прилипают к левому краю.
   - [ ] Android: то же.
   - [ ] Повернуть устройство — отступы слева/справа сохраняются.

2. **Pro-гейтинг**
   - [ ] Залогиниться Pro-аккаунтом (план с `has_extended_stats`).
   - [ ] Открыть «Статистика» → вкладка «Расширенная» активна, «Доступно в Pro» не показывается.
   - [ ] Перейти в «Подписки» и обратно на «Статистика» → по-прежнему «Расширенная» доступна, «Доступно в Pro» нет.
   - [ ] В __DEV__ в логе `[STATS] features` проверить `has_extended_stats: true`, `isPro: true`.

3. **Период**
   - [ ] Блок «Период» расположен сразу под KPI (под «Показатели» и двумя карточками).
   - [ ] Смена периода (день/неделя/месяц и т.д.) и prev/today/next обновляет KPI, топы и графики.
   - [ ] Топы по записям/доходу и графики отображаются без изменений и поломок.

4. **Общее**
   - [ ] Pull-to-refresh работает.
   - [ ] Переключение Базовая/Расширенная работает при Pro.
