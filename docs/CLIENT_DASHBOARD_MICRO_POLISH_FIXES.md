# ClientDashboard — Micro-polish исправления UI и логики избранного

## Дата: 2026-02-05

## Контекст
После предыдущего micro-polish остались 6 проблем, которые были исправлены без глобального рефактора, с сохранением зелёной темы и структуры секций.

## Список исправленных проблем

### 1. Выравнивание иконок в колонке "Действия" (Прошедшие записи)

**Проблема**: Иконки в колонке "Действия" визуально "разъехались" - стояли неравномерно/не в одной линии по строкам.

**Решение**:
- Обернул каждую иконку в `<div className="inline-flex items-center justify-center w-8 h-8">` для фиксированной ширины
- Изменил `gap-3` на `gap-2` для более компактного расположения
- Применил изменения как в основной таблице "Прошедшие записи", так и в модальном окне "Посмотреть все"

**Файлы**: `frontend/src/pages/ClientDashboard.jsx` (строки ~1195-1250, ~1902-1957)

---

### 2. Компактность секции "Избранные"

**Проблема**: Карточка "Избранные" занимала слишком много места по высоте.

**Решение**:
- Изменил layout с `grid grid-cols-3 gap-3` на `space-y-2` (вертикальный список)
- Переделал `renderFavoriteCard` из карточки в компактную строку с `flex items-center justify-between`
- Уменьшил padding: `p-3` вместо `p-4`, добавил `border-b` между строками
- Сердечко перенесено в правую часть строки (`flex-shrink-0`)
- Добавлен `hover:bg-gray-50` для интерактивности

**Файлы**: `frontend/src/pages/ClientDashboard.jsx` (строки ~1268-1294, ~430-465)

---

### 3. Ссылка на мастера в "Мои баллы"

**Проблема**: В таблице "Мои баллы" пропала активная ссылка на страницу мастера при наведении на имя.

**Решение**:
- Добавил поле `master_domain: Optional[str] = None` в схему `ClientLoyaltyPointsOut` (backend)
- Обновил endpoint `/api/client/loyalty/points` для получения `master.domain` из модели
- В компоненте `ClientLoyaltyPoints.jsx` добавил условный рендер:
  - Если `master_domain` есть → `<Link to={/domain/${master_domain}>` с зелёным hover-стилем
  - Если нет → обычный `<span>`
- Импортировал `Link` из `react-router-dom`

**Файлы**: 
- `backend/schemas.py` (строка ~2600)
- `backend/routers/client_loyalty.py` (строки ~77-95, ~98-116, ~180-187)
- `frontend/src/components/ClientLoyaltyPoints.jsx` (строки ~1-3, ~79-102)

---

### 4. Кнопка "История" в header секции "Мои баллы"

**Проблема**: Кнопка "История" была в каждой строке таблицы, что загромождало UI.

**Решение**:
- Удалил колонку "Действия" из таблицы `ClientLoyaltyPoints`
- Добавил кнопку "История" в header секции "Мои баллы" (рядом с "Посмотреть все")
- Создал state для модального окна истории: `showLoyaltyHistoryModal`, `loyaltyPoints`, `selectedLoyaltyMaster`
- Добавил функцию `handleShowLoyaltyHistory()`:
  - Загружает данные через `/api/client/loyalty/points`
  - Если один мастер → открывает сразу его историю
  - Если несколько → показывает список мастеров с кнопкой "Открыть"
- Модальное окно поддерживает навигацию: можно вернуться к списку мастеров кнопкой "← Назад"

**Файлы**: 
- `frontend/src/pages/ClientDashboard.jsx` (строки ~105-107, ~272-303, ~1296-1320, ~2032-2120)
- `frontend/src/components/ClientLoyaltyPoints.jsx` (удалены строки с модалкой и кнопкой "История" в строках)

---

### 5. Точный текст tooltip для dislike

**Проблема**: Tooltip был слишком длинным ("Не понравилось. Оценка поможет улучшить рекомендации.").

**Решение**:
- Заменил текст на: **"Не понравилось. Отобразится при следующем бронировании"**
- Применил изменения во всех местах:
  - Основная таблица "Прошедшие записи"
  - Модальное окно "Посмотреть все" (прошедшие)
- Сохранил компактный стиль: `max-w-[200px]`, `text-xs leading-tight`

**Файлы**: `frontend/src/pages/ClientDashboard.jsx` (строки ~1240-1247, ~1992-1999)

---

### 6. Синхронизация favorites между секциями

**Проблема**: После toggle сердечка в "Будущих" записях, сердечко в "Прошедших" у того же мастера не обновлялось (не становилось красным).

**Решение**:
- Привел все `itemId` к типу `Number` при добавлении/удалении из `favoriteMasterIds`:
  - В `handleFavoriteChange`: `const normalizedId = Number(itemId)`
  - При построении Set из API: `masterIds.add(Number(fav.master_id))`
- Применил `replace_all` для всех мест, где строится Set из favorites
- Это обеспечивает консистентность типов и корректную работу `Set.has()` при проверке состояния

**Файлы**: `frontend/src/pages/ClientDashboard.jsx` (строки ~277-287, ~255-261, ~380-386, ~618-624)
