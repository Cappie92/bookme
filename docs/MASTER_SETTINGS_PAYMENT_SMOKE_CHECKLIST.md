# Smoke-checklist: Настройки мастера и Способы оплаты (WEB)

## (1) Ручное подтверждение (POST-visit)

- [ ] **Включение manual confirm:** Переключить «Подтверждать каждую запись вручную» в настройках работы → сохранить.
- [ ] **Существующие pending:** Если до переключения были записи в статусе «На подтверждение» (AWAITING_CONFIRMATION) — они автоматически становятся подтверждёнными (COMPLETED). Pending-confirmations пуст или уменьшается.
- [ ] **Новые записи:** Создать новую запись после включения ручного режима, дождаться окончания времени визита → она появляется в pending-confirmations.
- [ ] **Старые подтверждённые:** Записи, уже подтверждённые/завершённые до переключения — не меняются.

## (2) Домен: только в «Управление сайтом»

- [ ] **Настройки работы:** В блоке «Редактирование профиля» / «Настройки работы» нет поля/инпута для редактирования домена.
- [ ] **Управление сайтом:** В блоке «Управление сайтом» при наличии `canCustomizeDomain` есть инпут для изменения URL домена. При сохранении настроек сайта домен сохраняется.

## (3) Способы оплаты

- [ ] **Оплата через DeDato:** Чекбокс «Принимаю оплату через систему DeDato» отключён (disabled), рядом отображается «Функция в разработке». Клик по чекбоксу не меняет состояние и не вызывает запросы.
- [ ] **Предоплата:** Текста «30 минут», «таймер», «автоотмена» нет. Текст заменён на: «В данный момент сервис не предоставляет функцию предоплаты. Осуществление предоплаты и контроль — на стороне мастера.»
- [ ] **Предоплата:** Чекбокс «Предоплата при бронировании» отключён (disabled).
- [ ] **Блок «Важно»:** Убраны пункты про «30 минут» и «автоматически отменяется». Оставлен пункт «Предоплата и её контроль осуществляются мастером вне платформы».

## (4) Регрессия

- [ ] Pre-visit подтверждения не затронуты.
- [ ] Owner filter для записей работает (отображаются только записи текущего мастера).
