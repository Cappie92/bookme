## 🔧 Что нужно сделать вам

### 1. Настройка переменных окружения

Создайте или обновите файл `backend/.env` со следующими переменными:

```env
# Robokassa Configuration
ROBOKASSA_MERCHANT_LOGIN=your_merchant_login
ROBOKASSA_PASSWORD_1=your_password_1
ROBOKASSA_PASSWORD_2=your_password_2
ROBOKASSA_IS_TEST=true

# URLs для уведомлений (для тестового режима)
ROBOKASSA_RESULT_URL=http://localhost:8000/api/payments/robokassa/result
ROBOKASSA_SUCCESS_URL=http://localhost:5173/payment/success
ROBOKASSA_FAIL_URL=http://localhost:5173/payment/failed
```

**Для боевого режима:**
```env
ROBOKASSA_IS_TEST=false
ROBOKASSA_RESULT_URL=https://api.your-domain.com/api/payments/robokassa/result
ROBOKASSA_SUCCESS_URL=https://app.your-domain.com/payment/success
ROBOKASSA_FAIL_URL=https://app.your-domain.com/payment/failed
```

**Где взять данные:**
1. Зарегистрируйтесь в [Robokassa](https://robokassa.ru/)
2. В личном кабинете найдите:
   - `MerchantLogin` - логин магазина
   - `Password #1` - пароль для генерации подписи запроса
   - `Password #2` - пароль для проверки подписи уведомлений

### 2. Применение миграций базы данных

Выполните миграции для создания таблицы `payments` и конвертации балансов:

```bash
cd backend
python3 -m alembic upgrade head
```

**Важно:** Миграция конвертирует все балансы с копеек на рубли. Убедитесь, что у вас есть резервная копия базы данных перед выполнением.

### 3. Настройка Robokassa в личном кабинете

#### 3.1. Настройка ResultURL (для уведомлений)

1. Войдите в личный кабинет Robokassa
2. Перейдите в раздел "Настройки" → "Технические настройки"
3. Укажите ResultURL:
   - **Тестовый режим:** `http://localhost:8000/api/payments/robokassa/result`
   - **Боевой режим:** `https://api.your-domain.com/api/payments/robokassa/result`

**Важно:** 
- ResultURL должен быть доступен из интернета (для боевого режима)
- Robokassa отправляет уведомления методом POST
- Эндпоинт не требует аутентификации, но проверяет подпись

#### 3.2. Настройка SuccessURL и FailURL

Эти URL указываются в каждом запросе на оплату, но можно также задать их по умолчанию в настройках Robokassa.

### 4. Проверка доступности ResultURL (для боевого режима)

Для боевого режима ResultURL должен быть доступен из интернета. Robokassa будет отправлять уведомления на этот адрес.

**Проверка:**
```bash
# Проверьте, что ваш сервер доступен из интернета
curl -X POST https://api.your-domain.com/api/payments/robokassa/result
```

### 5. Настройка тестовых данных Robokassa

В тестовом режиме Robokassa использует тестовые карты:
- **Успешная оплата:** `4111111111111111` (любая дата, любой CVV)
- **Отклоненная оплата:** `4111111111111112`

## 🆘 Возможные проблемы и решения

### Проблема: Уведомления от Robokassa не приходят

**Решение:**
1. Проверьте, что ResultURL доступен из интернета
2. Проверьте настройки в личном кабинете Robokassa
3. Проверьте логи сервера на наличие запросов
4. Убедитесь, что эндпоинт возвращает `OK{invoice_id}`

### Проблема: Неверная подпись

**Решение:**
1. Проверьте правильность `password_1` и `password_2`
2. Убедитесь, что используете правильный пароль для проверки (password_2 для ResultURL)
3. Проверьте формат суммы (должна быть с точкой, не запятой)

### Проблема: Платеж создан, но подписка не активирована

**Решение:**
1. Проверьте, что пользователь нажал кнопку активации
2. Проверьте логи на наличие ошибок при создании подписки
3. Проверьте баланс пользователя (должен быть пополнен)
