## 1. Получение данных от Robokassa

### 1.1. Регистрация в Robokassa

1. Перейдите на сайт [Robokassa](https://robokassa.ru/)
2. Зарегистрируйтесь или войдите в личный кабинет
3. Создайте новый магазин (если еще не создан)

### 1.2. Получение необходимых данных

В личном кабинете Robokassa вам понадобятся:

1. **Merchant Login** (Логин магазина)
   - Находится в разделе "Настройки магазина"
   - Пример: `my_shop_login`

2. **Password #1** (Пароль #1)
   - Используется для генерации подписи платежа
   - Находится в разделе "Технические настройки" → "Пароли"
   - Пример: `SMOsboVJA0919uMwBC4N`

3. **Password #2** (Пароль #2)
   - Используется для проверки уведомлений от Robokassa
   - Находится в том же разделе
   - Пример: `WX67emZei987hdTxiUZw`

4. **Тестовые данные** (для тестирования)
   - В тестовом режиме можно использовать тестовые карты
   - Тестовые карты: `4111111111111111` (любая дата, любой CVV)

## 2. Настройка переменных окружения

### 2.1. Создание файла .env

1. Перейдите в директорию `backend`:
   ```bash
   cd /Users/s.devyatov/DeDato/backend
   ```

2. Проверьте, существует ли файл `.env`:
   ```bash
   ls -la .env
   ```

3. Если файла нет, создайте его:
   ```bash
   touch .env
   ```

### 2.2. Добавление переменных Robokassa

Откройте файл `.env` в редакторе и добавьте следующие строки:

```env
# ============================================
# ROBOKASSA CONFIGURATION
# ============================================
# Логин магазина в Robokassa
ROBOKASSA_MERCHANT_LOGIN=your_merchant_login_here

# Пароль #1 (для генерации подписи платежа)
ROBOKASSA_PASSWORD_1=SMOsboVJA0919uMwBC4N

# Пароль #2 (для проверки уведомлений)
ROBOKASSA_PASSWORD_2=WX67emZei987hdTxiUZw

# Режим работы: true = тестовый, false = боевой
ROBOKASSA_IS_TEST=true

# URL для уведомлений от Robokassa (должен быть доступен из интернета)
ROBOKASSA_RESULT_URL=https://dedato.ru/api/payments/robokassa/result

# URL для успешной оплаты (куда перенаправить пользователя после оплаты)
ROBOKASSA_SUCCESS_URL=https://dedato.ru/payment/success

# URL для неудачной оплаты (куда перенаправить пользователя при ошибке)
ROBOKASSA_FAIL_URL=https://dedato.ru/payment/failed
```

### 2.3. Замена значений

**ВАЖНО:** Замените следующие значения на реальные:

1. `your_merchant_login_here` → ваш реальный Merchant Login из личного кабинета Robokassa
2. `SMOsboVJA0919uMwBC4N` → ваш реальный Password #1
3. `WX67emZei987hdTxiUZw` → ваш реальный Password #2

**Для тестирования на localhost:**

Если вы тестируете локально, используйте сервис для туннелирования (например, ngrok):

```env
# Для локального тестирования через ngrok
ROBOKASSA_RESULT_URL=https://your-ngrok-url.ngrok.io/api/payments/robokassa/result
ROBOKASSA_SUCCESS_URL=https://your-ngrok-url.ngrok.io/payment/success
ROBOKASSA_FAIL_URL=https://your-ngrok-url.ngrok.io/payment/failed
```

### 2.4. Проверка файла .env

Убедитесь, что файл `.env` не попадает в git (должен быть в `.gitignore`):

```bash
# Проверьте .gitignore
grep -q "^\.env$" .gitignore && echo "✅ .env в .gitignore" || echo "⚠️ Добавьте .env в .gitignore"
```

## 3. Настройка в личном кабинете Robokassa

### 3.1. Вход в личный кабинет

1. Перейдите на [https://auth.robokassa.ru/](https://auth.robokassa.ru/)
2. Войдите в личный кабинет
3. Выберите ваш магазин

### 3.2. Настройка Result URL

1. Перейдите в раздел **"Настройки магазина"** → **"Технические настройки"**
2. Найдите поле **"Result URL"** (URL для уведомлений)
3. Введите:
   ```
   https://dedato.ru/api/payments/robokassa/result
   ```
4. **Важно:** Этот URL должен быть доступен из интернета (не localhost)

### 3.3. Настройка Success URL

1. В том же разделе найдите поле **"Success URL"** (URL успешной оплаты)
2. Введите:
   ```
   https://dedato.ru/payment/success
   ```

### 3.4. Настройка Fail URL

1. Найдите поле **"Fail URL"** (URL неудачной оплаты)
2. Введите:
   ```
   https://dedato.ru/payment/failed
   ```

### 3.5. Настройка алгоритма хеширования

1. В разделе **"Технические настройки"** найдите **"Алгоритм хеширования"**
2. Выберите: **MD5** (или SHA256, в зависимости от ваших настроек)
3. **Важно:** Убедитесь, что алгоритм совпадает с тем, что используется в коде

### 3.6. Настройка паролей

1. В разделе **"Технические настройки"** → **"Пароли"**
2. Убедитесь, что:
   - **Password #1** совпадает с `ROBOKASSA_PASSWORD_1` в `.env`
   - **Password #2** совпадает с `ROBOKASSA_PASSWORD_2` в `.env`
3. Если пароли не совпадают, либо обновите их в Robokassa, либо обновите `.env`

### 3.7. Включение тестового режима

1. В разделе **"Настройки магазина"** найдите **"Тестовый режим"**
2. Включите тестовый режим для проверки интеграции
3. В тестовом режиме можно использовать тестовые карты

## 4. Проверка доступности URL

### 4.1. Проверка Result URL

Result URL должен быть доступен из интернета для получения уведомлений от Robokassa.

**Для проверки:**

1. Убедитесь, что ваш сервер запущен
2. Проверьте доступность URL:
   ```bash
   curl -X POST https://dedato.ru/api/payments/robokassa/result \
     -H "Content-Type: application/x-www-form-urlencoded" \
     -d "OutSum=100&InvId=test&SignatureValue=test"
   ```

3. Должен вернуться ответ (даже если с ошибкой, главное - что URL доступен)

### 4.2. Проверка Success URL и Fail URL

Эти URL должны быть доступны для пользователей:

```bash
# Проверка Success URL
curl https://dedato.ru/payment/success

# Проверка Fail URL
curl https://dedato.ru/payment/failed
```

### 4.3. Использование ngrok для локального тестирования

Если вы тестируете локально, используйте ngrok:

1. Установите ngrok:
   ```bash
   brew install ngrok  # для macOS
   ```

2. Запустите туннель:
   ```bash
   ngrok http 8000  # если ваш бэкенд на порту 8000
   ```

3. Скопируйте HTTPS URL (например: `https://abc123.ngrok.io`)

4. Обновите `.env`:
   ```env
   ROBOKASSA_RESULT_URL=https://abc123.ngrok.io/api/payments/robokassa/result
   ROBOKASSA_SUCCESS_URL=https://abc123.ngrok.io/payment/success
   ROBOKASSA_FAIL_URL=https://abc123.ngrok.io/payment/failed
   ```

5. Обновите настройки в личном кабинете Robokassa

## 5. Тестирование интеграции

### 5.1. Запуск бэкенда

1. Перейдите в директорию `backend`:
   ```bash
   cd /Users/s.devyatov/DeDato/backend
   ```

2. Активируйте виртуальное окружение (если используется):
   ```bash
   source venv/bin/activate  # или ваш путь к venv
   ```

3. Запустите сервер:
   ```bash
   python3 -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
   ```

4. Убедитесь, что сервер запустился без ошибок

### 5.2. Проверка эндпоинтов

#### 5.2.1. Проверка инициализации платежа за подписку

```bash
curl -X POST http://localhost:8000/api/payments/subscription/init \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "plan_id": 1,
    "duration_months": 1,
    "payment_period": "month",
    "enable_auto_renewal": false
  }'
```

**Ожидаемый ответ:**
```json
{
  "payment_url": "https://auth.robokassa.ru/Merchant/Index.aspx?...",
  "payment_id": 1,
  "invoice_id": "INV-1234567890-1"
```

```bash
curl -X POST http://localhost:8000/api/payments/deposit/init \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "amount": 1000
  }'
```

### 5.3. Тестирование полного цикла оплаты

1. **Откройте личный кабинет мастера:**
   - Войдите в систему как мастер
   - Перейдите в раздел "Мой тариф" или откройте модальное окно подписки

2. **Выберите тариф:**
   - Выберите план подписки
   - Выберите продолжительность (1, 3, 6 или 12 месяцев)
   - При необходимости включите автопродление

3. **Нажмите "Перейти к оплате":**
   - Должно произойти перенаправление на страницу Robokassa
   - URL должен начинаться с `https://auth.robokassa.ru/`

4. **Оплата в тестовом режиме:**
   - Используйте тестовую карту: `4111111111111111`
   - Любая дата в будущем (например: 12/25)
   - Любой CVV (например: 123)
   - Любое имя держателя

5. **Проверка успешной оплаты:**
   - После успешной оплаты вы должны быть перенаправлены на `/payment/success`
   - Через 10 секунд автоматический редирект на `/master/tariff`
   - В разделе "Мой тариф" должна появиться кнопка "Активировать подписку"

6. **Проверка неудачной оплаты:**
   - Отмените оплату на странице Robokassa
   - Вы должны быть перенаправлены на `/payment/failed`
   - Через 10 секунд автоматический редирект обратно в личный кабинет

### 5.4. Проверка уведомлений от Robokassa

1. После успешной оплаты Robokassa отправит уведомление на Result URL
2. Проверьте логи бэкенда - должны быть записи о получении уведомления
3. Проверьте базу данных - платеж должен быть обновлен со статусом `paid`

### 5.5. Проверка активации подписки

1. После успешной оплаты перейдите в раздел "Мой тариф"
2. Нажмите кнопку "Активировать подписку"
3. Подписка должна стать активной
4. Проверьте, что баланс был пополнен и зарезервирован для подписки

## 7. Решение проблем

### 7.1. Ошибка "Invalid signature"

**Причина:** Неправильная подпись платежа

**Решение:**
1. Проверьте, что `ROBOKASSA_PASSWORD_1` совпадает с Password #1 в Robokassa
2. Проверьте, что алгоритм хеширования совпадает
3. Убедитесь, что сумма платежа передается корректно

### 7.2. Ошибка "Payment not found"

**Причина:** Платеж не найден в базе данных

**Решение:**
1. Проверьте, что `InvoiceID` передается корректно
2. Проверьте, что платеж был создан перед перенаправлением на Robokassa
3. Проверьте логи бэкенда для деталей

### 7.3. Result URL недоступен

**Причина:** Robokassa не может отправить уведомление

**Решение:**
1. Убедитесь, что URL доступен из интернета (не localhost)
2. Используйте ngrok для локального тестирования
3. Проверьте, что сервер запущен и доступен
4. Проверьте firewall и настройки безопасности

### 7.4. Пользователь не перенаправляется после оплаты

**Причина:** Неправильные Success URL или Fail URL

**Решение:**
1. Проверьте настройки в личном кабинете Robokassa
2. Убедитесь, что URL указаны правильно
3. Проверьте, что фронтенд доступен по указанным URL

### 7.5. Подписка не активируется после оплаты

**Причина:** Проблема с обработкой уведомления или активацией

**Решение:**
1. Проверьте логи бэкенда при получении уведомления
2. Проверьте, что платеж обновлен со статусом `paid`
3. Проверьте, что баланс пополнен
4. Проверьте, что резерв создан для подписки

### 7.6. Ошибка при конвертации суммы

**Причина:** Неправильный формат суммы

**Решение:**
1. Убедитесь, что сумма передается как число (float)
2. Проверьте, что сумма в рублях (не копейках)
3. Проверьте логи для деталей ошибки
