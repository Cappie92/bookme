# План реализации системы уровней подписки для мастеров

## Обновленный план действий

### Этап 1: Расширение базы данных

1.1. Создать модель `SubscriptionPlan`:
   - `id`, `name` (Free, Basic, Pro, Premium)
   - `subscription_type` (MASTER для мастеров)
   - `price_monthly`, `price_yearly`
   - `features` (JSON) — список включенных функций:
     ```json
     {
       "can_customize_domain": false,
       "can_add_page_modules": false,
       "has_finance_access": false,
       "has_extended_stats": false,
       "max_page_modules": 0,
       "stats_retention_days": 30
     }
     ```
   - `limits` (JSON) — квоты (bookings, services, etc.)
   - `is_active`, `display_order`

1.2. Создать модель `SubscriptionAddon`:
   - `id`, `name`, `description`
   - `subscription_type` (MASTER)
   - `price_monthly`, `price_yearly`
   - `feature_key` — ключ функции (например, "extra_page_module")
   - `is_active`

1.3. Создать модель `MasterPageModule`:
   - `id`, `master_id`, `module_type` (text, image, video, booking_form, etc.)
   - `position` (order), `config` (JSON), `is_active`

1.4. Обновить модель `Subscription`:
   - Добавить `plan_id` (ForeignKey к `SubscriptionPlan`)
   - Добавить `addons` (JSON) — список подключенных опций
   - Сохранить обратную совместимость с количественными параметрами

1.5. Создать миграцию Alembic

### Этап 2: Backend API — управление планами

2.1. Создать роутер `subscription_plans.py`:

   - `PUT /api/admin/subscription-plans/{id}` — обновить план
   - `DELETE /api/admin/subscription-plans/{id}` — удалить план
   - `GET /api/subscription-plans/available` — доступные планы для мастера

2.2. Создать роутер `subscription_addons.py`:

2.3. Обновить `subscriptions.py`:

   - Сохранить совместимость с калькулятором

2.4. Создать схемы Pydantic:
   - `SubscriptionPlanCreate`, `SubscriptionPlanUpdate`, `SubscriptionPlanOut`

### Этап 3: Backend — проверка доступа к функциям

3.1. Создать утилиту `subscription_features.py`:
   - `check_feature_access(db, user_id, feature_key)` — проверка доступа к функции
   - `get_master_features(db, user_id)` — получить все доступные функции мастера
   - `can_customize_domain(db, user_id)` — проверка возможности изменения домена
   - `can_add_page_module(db, user_id)` — проверка возможности добавления модулей
   - `has_finance_access(db, user_id)` — проверка доступа к финансам
   - `has_extended_stats(db, user_id)` — проверка расширенной статистики
   - `get_max_page_modules(db, user_id)` — получить максимальное количество модулей

3.2. Обновить `subscription_limits.py`:
   - Интегрировать проверки из `subscription_features.py`
   - Добавить проверку лимитов из `SubscriptionPlan`

3.3. Обновить `routers/master.py`:
   - В `update_master_profile` добавить проверку `can_customize_domain` перед изменением домена
   - Вернуть 403, если функция недоступна

3.4. Обновить `routers/domain.py`:
   - Добавить проверку `can_add_page_module` при добавлении модулей на страницу
   - Вернуть информацию о доступных модулях в `get_subdomain_info`
