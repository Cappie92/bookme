# Копирование базы данных на сервер

## ✅ Правильный способ использования

Скрипт `copy_db_to_server.sh` нужно запускать **ЛОКАЛЬНО** (на вашем компьютере), а не на сервере!

### Шаг 1: Убедитесь, что вы в корневой директории проекта
```bash
cd /Users/s.devyatov/DeDato
```

### Шаг 2: Запустите скрипт локально
```bash
./copy_db_to_server.sh
```

Скрипт автоматически:
1. ✅ Проверит подключение к серверу
2. ✅ Остановит backend на сервере
3. ✅ Создаст бэкап текущей БД на сервере
4. ✅ Скопирует вашу локальную БД на сервер
5. ✅ Запустит backend на сервере
6. ✅ Проверит, что API работает

## ❌ Неправильно

Не запускайте скрипт на сервере! Скрипт должен копировать файл **С** локального компьютера **НА** сервер.

## 🔧 Альтернатива: ручное копирование

Если скрипт не работает, можно скопировать БД вручную:

```bash
# 1. Остановите backend на сервере
ssh root@193.160.208.206 'cd /home/root/dedato && docker-compose -f docker-compose.prod.yml stop backend'

# 2. Создайте бэкап текущей БД (на сервере)
ssh root@193.160.208.206 'cd /home/root/dedato && mv backend/bookme.db backend/bookme.db.backup.$(date +%Y%m%d_%H%M%S)'

# 3. Скопируйте локальную БД на сервер (из локальной директории!)
scp backend/bookme.db root@193.160.208.206:/home/root/dedato/backend/bookme.db

# 4. Запустите backend на сервере
ssh root@193.160.208.206 'cd /home/root/dedato && docker-compose -f docker-compose.prod.yml start backend'
```

## 📝 Важно

- Скрипт должен запускаться **из локальной директории проекта**
- Локальная БД должна существовать: `backend/bookme.db`
- У вас должен быть SSH доступ к серверу (обычно через SSH ключи)
