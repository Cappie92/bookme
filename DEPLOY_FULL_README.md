# 🚀 Полный деплой локальной версии на продакшн

## Описание

Скрипт `deploy_full.sh` полностью заменяет все файлы на продакшн сервере локальной версией проекта.

## ⚠️ Важные моменты

- **База данных**: Скрипт сохраняет существующую БД в бэкап перед заменой файлов
- **Миграции**: Все миграции применяются автоматически
- **Контейнеры**: Полная пересборка и перезапуск всех контейнеров

## 🎯 Что делает скрипт

1. ✅ Создает бэкап базы данных
2. ✅ Останавливает все контейнеры на сервере
3. ✅ Копирует все backend файлы (исключая временные файлы и БД)
4. ✅ Копирует все frontend файлы (исключая node_modules и dist)
5. ✅ Копирует конфигурационные файлы (docker-compose.prod.yml и др.)
6. ✅ Сохраняет существующую БД на сервере (если есть)
7. ✅ Применяет все миграции Alembic
8. ✅ Пересобирает Docker контейнеры
9. ✅ Запускает контейнеры
10. ✅ Проверяет статус и логи
11. ✅ Проверяет доступность API и Frontend

## 📋 Требования

- Доступ по SSH к серверу `root@193.160.208.206`
- Установленный `rsync` (обычно уже установлен)
- Запуск из корневой директории проекта

## 🚀 Использование

**ВАЖНО:** Скрипт запускается **локально** на вашей машине, а не на сервере!

```bash
# На вашей локальной машине, в корневой директории проекта:
cd /Users/s.devyatov/DeDato
./deploy_full.sh
```

Скрипт автоматически:
- Подключится к серверу по SSH
- Скопирует все файлы
- Выполнит все необходимые команды на сервере

**Не нужно** заходить на сервер или переходить в папку проекта на сервере - все делается автоматически!

## 📍 Результат

После выполнения:
- Все файлы на сервере будут заменены локальными версиями
- База данных будет сохранена и применены миграции
- Сервисы будут доступны по адресам:
  - Frontend: http://193.160.208.206:5173
  - API: http://193.160.208.206:8000
  - API Docs: http://193.160.208.206:8000/docs

## 🔄 Откат

Если что-то пошло не так:

1. Бэкап БД сохранен локально в `backups/bookme_backup_YYYYMMDD_HHMMSS.db`
2. Старая БД на сервере сохранена в `backend/bookme.db.backup.YYYYMMDD_HHMMSS`
3. Можно восстановить БД из бэкапа:
   ```bash
   scp backups/bookme_backup_YYYYMMDD_HHMMSS.db root@193.160.208.206:/home/root/dedato/backend/bookme.db
   ssh root@193.160.208.206 "cd /home/root/dedato && docker-compose -f docker-compose.prod.yml restart backend"
   ```
