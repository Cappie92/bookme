# Объяснение портов в мобильном приложении

## Важно: это РАЗНЫЕ порты для РАЗНЫХ сервисов!

### Порт 8001 - Backend API (FastAPI)
- **Что это:** Ваш backend сервер (FastAPI)
- **Где запускается:** Отдельно, в папке `backend/` или через Docker
- **Для чего:** Обработка API запросов от мобильного приложения
- **Проверка:** `lsof -ti:8001` должен показать процесс

### Порт 8081/8082 - Metro Bundler (Expo Dev Server)
- **Что это:** Сервер разработки Expo для мобильного приложения
- **Где запускается:** Когда вы запускаете `npx expo start`
- **Для чего:** Отдает JavaScript код приложения на устройство/эмулятор
- **Это нормально:** Expo автоматически выбирает свободный порт (8081, 8082, и т.д.)

## Как это работает вместе:

```
┌─────────────────┐         ┌──────────────────┐         ┌──────────────┐
│  Мобильное      │         │  Metro Bundler   │         │  Backend API │
│  приложение     │◄───────►│  (порт 8081)     │         │  (порт 8001) │
│  (Expo Go)      │         │                  │         │              │
└─────────────────┘         └──────────────────┘         └──────────────┘
                                      │                            ▲
                                      │                            │
                                      └────────────────────────────┘
                                      API запросы (login, bookings, etc.)
```

## Проверка что все работает:

### 1. Backend должен быть запущен на порту 8001:
```bash
# Проверить что backend запущен
lsof -ti:8001

# Если не запущен, запустите:
cd /Users/s.devyatov/DeDato/backend
# или через Docker
```

### 2. Metro bundler должен быть запущен на 8081/8082:
```bash
cd /Users/s.devyatov/DeDato/mobile
npx expo start
# Должен показать порт 8081 или 8082 - это нормально!
```

### 3. В .env файле должен быть правильный API_URL:
```bash
cd /Users/s.devyatov/DeDato/mobile
cat .env
# Должно быть что-то вроде:
# API_URL=http://localhost:8001
# или для эмулятора:
# API_URL=http://10.0.2.2:8001  (Android)
# API_URL=http://localhost:8001 (iOS симулятор)
```

## Проблема с эмулятором Android

Если приложение не запускается на Android эмуляторе, проблема может быть в API_URL:

### Для Android эмулятора:
- `localhost` не работает!
- Нужно использовать `10.0.2.2` вместо `localhost`
- Или использовать IP адрес вашего Mac в локальной сети

### Проверьте .env файл:
```bash
cd /Users/s.devyatov/DeDato/mobile
cat .env
```

Если там `API_URL=http://localhost:8001`, для Android эмулятора нужно изменить на:
```
API_URL=http://10.0.2.2:8001
```

Или использовать IP адрес вашего Mac:
```
API_URL=http://192.168.1.XXX:8001
```

## Резюме:

✅ **Порт 8081/8082** - это нормально для Metro bundler
✅ **Порт 8001** - должен быть запущен backend отдельно
✅ **Проблема скорее всего в API_URL** для Android эмулятора
