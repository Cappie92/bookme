# 🧪 Начало тестирования - Пошаговая инструкция

## ✅ Шаг 1: Проверка .env файла

Файл `.env` уже существует в `mobile/`. Проверьте его содержимое:

```bash
cat mobile/.env
```

Должно быть:
```env
API_URL=http://localhost:8000
```

**Если тестируете на реальном устройстве/эмуляторе**, замените на IP вашей машины:
```env
API_URL=http://192.168.1.100:8000
```

Чтобы узнать IP вашей машины:
- **macOS/Linux:** `ifconfig | grep "inet " | grep -v 127.0.0.1`
- **Windows:** `ipconfig` (ищите IPv4 адрес)

## ✅ Шаг 3: Запуск backend

**Важно:** Backend должен быть запущен перед тестированием мобильного приложения!

В отдельном терминале:

```bash
cd backend
python3 -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

Должно появиться:
```
INFO:     Uvicorn running on http://0.0.0.0:8000
```

## ✅ Шаг 4: Запуск мобильного приложения

В новом терминале:

```bash
cd mobile
npm start
```

Вы увидите:
- QR-код для сканирования
- Опции: `Press i │ open iOS simulator`, `Press a │ open Android emulator`

### Варианты запуска:

#### A) Expo Go на телефоне (рекомендуется)
1. Установите **Expo Go** из App Store / Google Play
2. Откройте Expo Go
3. Отсканируйте QR-код из терминала
4. Приложение загрузится на телефон

#### B) iOS Simulator (только macOS)
```bash
# В терминале с запущенным npm start нажмите:
i
```

#### C) Android Emulator
```bash
# В терминале с запущенным npm start нажмите:
a
```

#### D) Веб-браузер (для быстрого тестирования UI)
```bash
cd mobile
npm run web
```

## 🐛 Решение проблем

### Проблема: "Network Error" при входе

**Причина:** Backend не запущен или недоступен

**Решение:**
1. Проверьте, что backend запущен: `http://localhost:8000/docs` должен открываться
2. Если тестируете на устройстве, используйте IP вместо localhost в `.env`
3. Проверьте, что порт 8000 не заблокирован

### Проблема: "401 Unauthorized"

**Причина:** Неверные данные для входа или проблема с токеном

**Решение:**
1. Проверьте телефон и пароль
2. Попробуйте зарегистрировать нового пользователя
3. Проверьте логи Metro bundler - должен быть `🔑 [API] Токен добавлен`

### Проблема: Приложение не загружается

**Решение:**
1. Очистите кэш: `npm start -- --clear`
2. Переустановите зависимости: `rm -rf node_modules && npm install`
3. Проверьте версию Node.js: `node --version` (должна быть 18+)

### Проблема: Экран мастера не открывается

**Решение:**
1. Проверьте, что вошли как мастер (роль `MASTER` или `master`)
2. Проверьте логи - должны быть запросы к `/api/master/...`
3. Если ошибка 404 - проверьте, что Master профиль создан в БД
