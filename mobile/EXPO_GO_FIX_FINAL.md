# Финальное исправление для Expo Go

## Что я исправил

✅ Закомментировал кастомные `bundleIdentifier` и `package` в `app.config.ts` - они не поддерживаются Expo Go

## Что нужно сделать

### Шаг 1: Перезапустить Expo

```bash
cd /Users/s.devyatov/DeDato/mobile
# Остановите текущий Expo (Ctrl+C)
npx expo start --tunnel --clear
```

### Шаг 2: Попробовать подключиться

1. **В Expo Go нажмите на сервер** или отсканируйте QR-код
2. **Или используйте "Enter URL manually"** с URL из терминала

### Шаг 3: Проверить логи

В терминале Metro должны появиться:
- Запросы на `/index.bundle`
- Логи компиляции
- Ошибки (если есть)

## Альтернатива: Использовать эмулятор (рекомендуется)

Для Maestro тестов лучше использовать эмулятор:

### Android:
```bash
npx expo start --tunnel --clear
# Нажмите 'a'
```

### iOS:
```bash
npx expo start --tunnel --clear
# Нажмите 'i'
```

Эмуляторы работают надежнее и не требуют Expo Go.

## Если все еще не работает

Проверьте в терминале Metro:
- Есть ли ошибки компиляции
- Появляются ли запросы при подключении
- Работает ли bundle (попробуйте открыть URL в браузере)
