# 🔧 Troubleshooting: Network Error

## Проблема: "Network Error" при попытке входа

### Возможные причины и решения

---

## 1. ✅ CORS в Backend (ИСПРАВЛЕНО)

**Проблема:** Backend блокировал запросы от мобильного приложения из-за CORS.

**Решение:** Обновлён `backend/main.py` — теперь в development режиме разрешены все origins.

**Проверка:**
- Убедись, что backend запущен в development режиме
- Или установи переменную окружения: `ENVIRONMENT=development`

---

## 2. 🔍 Проверка API_URL

**Проблема:** Неправильный или пустой API_URL в `.env`.

**Решение:**

1. Открой `mobile/.env` и проверь:
   ```env
   API_URL=http://192.168.0.194:8001
   ```

2. **Важно:** Используй IP адрес твоего компьютера, а не `localhost` или `127.0.0.1`
   - `localhost` не работает с реального телефона
   - Нужен IP адрес в локальной сети (например: `192.168.0.194`)

3. **Как узнать IP адрес:**
   ```bash
   # macOS/Linux
   ifconfig | grep "inet " | grep -v 127.0.0.1
   
   # Или проще:
   ipconfig getifaddr en0  # macOS
   ```

4. **Проверь порт:** Убедись, что backend запущен на порту `8001` (или измени порт в `.env`)

---

## 3. 🌐 Backend не запущен или недоступен

**Проблема:** Backend не запущен или недоступен по указанному адресу.

**Решение:**

1. **Запусти backend:**
   ```bash
   cd backend
   # Убедись, что backend запущен на порту 8001 (или измени порт)
   uvicorn main:app --host 0.0.0.0 --port 8001 --reload
   ```

2. **Проверь доступность:**
   - Открой в браузере на компьютере: `http://192.168.0.194:8001/health`
   - Должен вернуться: `{"status": "healthy", "service": "DeDato API"}`

3. **Проверь firewall:**
   - Убедись, что firewall не блокирует порт 8001
   - macOS: Системные настройки → Безопасность → Firewall

---

## 4. 📱 Телефон и компьютер в разных сетях

**Проблема:** Телефон и компьютер подключены к разным Wi-Fi сетям.

**Решение:**

1. **Убедись, что оба устройства в одной Wi-Fi сети**

2. **Или используй tunnel режим:**
   ```bash
   cd mobile
   npm start -- --tunnel
   ```
   Это создаст публичный URL через Expo сервис (работает даже в разных сетях).

---

## 5. 🔍 Детальная диагностика

**Добавлено улучшенное логирование ошибок:**

1. Открой консоль Metro bundler (где запущен `npm start`)
2. Попробуй войти в приложение
3. Посмотри логи — там будет детальная информация:
   - URL запроса
   - Код ошибки
   - Сообщение об ошибке

**Пример логов:**
```
Network Error: {
  message: "Network Error",
  url: "/api/auth/login",
  baseURL: "http://192.168.0.194:8001",
  fullURL: "http://192.168.0.194:8001/api/auth/login",
  code: "ECONNREFUSED"
}
```

---

## 6. ✅ Чеклист для проверки

- [ ] Backend запущен и доступен по `http://<IP>:8001/health`
- [ ] API_URL в `mobile/.env` указывает на правильный IP и порт
- [ ] Телефон и компьютер в одной Wi-Fi сети
- [ ] Firewall не блокирует порт 8001
- [ ] Backend запущен с `--host 0.0.0.0` (не `localhost`)
- [ ] CORS настроен правильно (обновлён `backend/main.py`)

---

## 7. 🧪 Тестовый запрос

Попробуй сделать тестовый запрос из терминала:

```bash
# С компьютера
curl http://192.168.0.194:8001/health

# Должен вернуться:
# {"status":"healthy","service":"DeDato API"}
```

Если не работает — проблема в backend или сети.

---

## 8. 📝 Быстрое решение

Если ничего не помогает, попробуй:

1. **Используй tunnel режим:**
   ```bash
   cd mobile
   npm start -- --tunnel
   ```

2. **Или используй локальную сборку:**
   ```bash
   cd mobile
   npx expo run:android
   ```
   Это создаст development build, который может обойти некоторые проблемы с сетью.

---

## 💡 Советы

- **Для разработки:** Используй tunnel режим — он работает даже в разных сетях
- **Для тестирования:** Убедись, что backend доступен из браузера на телефоне
- **Для production:** Настрой правильный CORS и используй домен вместо IP
