# Исправление ошибки "Failed to download remote update"

## Ошибка
```
Uncaught Error: java.io.IOException: Failed to download remote update
```

Это означает, что Expo Go не может загрузить JavaScript bundle с Metro bundler.

## Решения (попробуйте по порядку)

### Решение 1: Проверить что Metro bundler работает

В терминале где запущен `npx expo start --tunnel` должны быть:
- ✅ Логи Metro bundler
- ✅ Нет ошибок компиляции
- ✅ Показан URL для подключения

Если есть ошибки компиляции - исправьте их.

### Решение 2: Очистить кеш полностью

```bash
cd /Users/s.devyatov/DeDato/mobile
# Остановите Expo (Ctrl+C)

# Очистить кеш Metro
npx expo start --tunnel --clear

# Или полная очистка
rm -rf node_modules/.cache
rm -rf .expo
npx expo start --tunnel --clear
```

### Решение 3: Проверить сеть/туннель

Проблема может быть в туннеле. Попробуйте:

1. **Перезапустить с туннелем:**
   ```bash
   npx expo start --tunnel --clear
   ```

2. **Или попробовать LAN режим** (если устройства в одной сети):
   ```bash
   npx expo start --lan --clear
   ```

### Решение 4: Проверить логи в терминале

При попытке подключения в терминале Expo должны появиться:
- Запросы на `/index.bundle`
- Ошибки (если есть)

**Если есть ошибки** - скопируйте их полностью.

### Решение 5: Использовать эмулятор (рекомендуется для тестирования)

Для Maestro тестов лучше использовать эмулятор - он работает надежнее:

#### Android эмулятор:
```bash
npx expo start --tunnel --clear
# Нажмите 'a' в терминале
```

#### iOS симулятор:
```bash
npx expo start --tunnel --clear
# Нажмите 'i' в терминале
```

Эмуляторы:
- ✅ Не требуют Expo Go
- ✅ Работают быстрее
- ✅ Всегда доступны для Maestro
- ✅ Не зависят от сети/туннеля

### Решение 6: Проверить что backend доступен

Убедитесь что backend работает:
```bash
curl http://localhost:8001/docs
```

Если backend не работает - запустите его.

## Что проверить в терминале Expo

При запуске `npx expo start --tunnel --clear` должны быть:
- ✅ Metro bundler запущен
- ✅ Показан QR-код
- ✅ Показан URL (exp://...)
- ✅ Нет ошибок компиляции TypeScript/JavaScript

Если есть ошибки - исправьте их перед подключением.

## Рекомендация

Для E2E тестов с Maestro **лучше использовать эмулятор/симулятор**, а не физическое устройство через Expo Go:
- Эмуляторы работают надежнее
- Не требуют Expo Go
- Всегда доступны для Maestro
- Быстрее для тестирования
