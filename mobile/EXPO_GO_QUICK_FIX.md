# Быстрое исправление проблемы Expo Go

## Что я исправил

✅ Отключил `newArchEnabled` в `app.json` (может вызывать проблемы с Expo Go)

## Что нужно сделать сейчас

### Шаг 1: Очистить кеш и перезапустить Expo

```bash
cd /Users/s.devyatov/DeDato/mobile
# Остановите текущий Expo (Ctrl+C)
npx expo start --tunnel --clear
```

Флаг `--clear` очистит кеш Metro bundler.

### Шаг 2: Попробуйте подключиться

1. **В Expo Go нажмите на сервер "DeDato on MacBook-Air-4.local"**

2. **Или используйте "Enter URL manually":**
   - Скопируйте URL из терминала Expo
   - Вставьте в Expo Go

### Шаг 3: Проверьте логи в терминале

При нажатии на сервер в Expo Go, в терминале должны появиться:
- Запросы на загрузку bundle
- Ошибки (если есть)

**Если есть ошибки** - скопируйте их и покажите мне.

## Альтернатива: Использовать эмулятор для тестирования

Для Maestro тестов лучше использовать эмулятор - он работает надежнее:

### Android эмулятор:
```bash
npx expo start --tunnel --clear
# Нажмите 'a'
```

### iOS симулятор:
```bash
npx expo start --tunnel --clear
# Нажмите 'i'
```

Эмуляторы:
- ✅ Не требуют Expo Go
- ✅ Работают быстрее
- ✅ Всегда доступны для Maestro
- ✅ Не зависят от сети

## Если все еще не работает

Проверьте что в терминале Expo:
- ✅ Нет ошибок компиляции
- ✅ Metro bundler запущен
- ✅ Показан QR-код и URL

Если есть ошибки - покажите их, и я помогу исправить.
