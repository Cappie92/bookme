# Переключение на Expo Go режим

## Проблема

Expo запущен в режиме development build, но development build не установлен.

## Решение: Переключиться на Expo Go

В терминале где запущен Expo нажмите **`s`** - это переключит на Expo Go режим.

После переключения:
- В терминале должно появиться "Using Expo Go"
- Затем нажмите **`i`** для iOS симулятора

## Альтернатива: Запустить сразу в Expo Go режиме

Остановите текущий Expo (Ctrl+C) и запустите с флагом `--go`:

```bash
cd /Users/s.devyatov/DeDato/mobile
npx expo start --go --clear
```

**Примечание:** `--tunnel` не нужен для локального симулятора, он нужен только для физических устройств.

Затем нажмите **`i`** для iOS симулятора.

## Важно

В режиме Expo Go некоторые функции могут не работать (например, кастомные нативные модули), но для базового тестирования этого достаточно.

## Важно: API_URL для iOS симулятора

Для iOS симулятора в `mobile/.env` должен быть:
```
API_URL=http://localhost:8001
```

**Не используйте** `10.0.2.2` - это только для Android эмулятора.

## После успешного запуска

Приложение должно открыться в симуляторе, и вы сможете:
- Тестировать вручную
- Запускать Maestro тесты
