# 🔍 Как посмотреть логи Network Error

## Проблема: Логи не видны в консоли

Логи появятся **только когда ты попробуешь войти в приложение**.

---

## 📋 Пошаговая инструкция

### 1. Убедись, что приложение открыто на телефоне

- Отсканируй QR код из консоли
- Или нажми `a` для Android / `i` для iOS симулятора

### 2. Попробуй войти в приложение

- Введи номер телефона и пароль
- Нажми "Войти"

### 3. Смотри логи в консоли Metro bundler

**Сразу после нажатия "Войти" в консоли должны появиться строки:**

```
Network Error: {
  message: "...",
  url: "/api/auth/login",
  baseURL: "http://192.168.0.194:8001",
  fullURL: "http://192.168.0.194:8001/api/auth/login",
  code: "..."
}
```

**Или:**

```
✅ API_URL настроен: http://192.168.0.194:8001
Login error details: { ... }
```

---

## ⚠️ Важно: Development Build vs Expo Go

Я вижу, что у тебя запущен **development build**, а не Expo Go.

**Для быстрой разработки лучше использовать Expo Go:**

1. В консоли Metro bundler нажми: `s` (switch to Expo Go)
2. Или перезапусти с явным указанием:
   ```bash
   npm start -- --go
   ```

**Expo Go проще для разработки и не требует сборки.**

---

## 🔧 Если логи всё равно не появляются

### Вариант 1: Проверь в DevTools

1. В приложении на телефоне: встряхни телефон
2. Выбери "Debug Remote JS" или "Open Debugger"
3. Открой консоль браузера (Chrome DevTools)
4. Там будут все логи

### Вариант 2: Добавь console.log в код

Временно добавь в `mobile/app/login.tsx` перед `handleLogin`:

```typescript
console.log('🔍 Attempting login with:', { phone, API_URL: env.API_URL });
```

Это поможет увидеть, что происходит.

---

## 📱 Быстрая проверка

1. **Открой приложение на телефоне** (отсканируй QR код)
2. **Попробуй войти** (введи данные и нажми "Войти")
3. **Сразу смотри в консоль Metro bundler** — логи должны появиться

---

## 💡 Совет

Если используешь **development build**, убедись, что он собран с правильными настройками. Для быстрой разработки лучше использовать **Expo Go** (нажми `s` в консоли).
