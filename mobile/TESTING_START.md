# 🧪 Инструкция по началу тестирования мобильной версии

## 📋 Подготовка к тестированию

### 1. Настройка переменных окружения

Создайте файл `.env` в папке `mobile/` (если его еще нет):

```bash
cd mobile
touch .env
```

Добавьте в файл `.env` следующее содержимое:

```env
API_URL=http://localhost:8000
```

**Важно:** 
- Для локального тестирования используйте `http://localhost:8000` (если backend запущен локально)
- Для тестирования с реальным устройством/эмулятором используйте IP-адрес вашей машины: `http://192.168.x.x:8000`
- Для тестирования с продакшн сервером: `https://dedato.ru`

### 2. Установка зависимостей

Убедитесь, что все зависимости установлены:

```bash
cd mobile
npm install
```

### 3. Проверка конфигурации

Проверьте, что:
- ✅ Файл `.env` существует в `mobile/`
- ✅ `API_URL` указан в `.env`
- ✅ Backend запущен и доступен по указанному адресу

## 🚀 Запуск приложения

### Вариант 1: Expo Go (рекомендуется для быстрого старта)

1. Установите приложение **Expo Go** на ваш телефон:
   - [iOS App Store](https://apps.apple.com/app/expo-go/id982107779)
   - [Google Play](https://play.google.com/store/apps/details?id=host.exp.exponent)

2. Запустите Metro bundler:
   ```bash
   cd mobile
   npm start
   ```

3. Отсканируйте QR-код:
   - **iOS**: Откройте камеру и нажмите на уведомление
   - **Android**: Откройте Expo Go и нажмите "Scan QR code"

### Вариант 2: iOS Simulator (только для macOS)

```bash
cd mobile
npm run ios
```

**Требования:**
- macOS
- Xcode установлен
- iOS Simulator доступен

### Вариант 3: Android Emulator

```bash
cd mobile
npm run android
```

**Требования:**
- Android Studio установлен
- Android Emulator запущен

### Вариант 4: Веб-браузер (для быстрого тестирования UI)

```bash
cd mobile
npm run web
```

Приложение откроется в браузере по адресу `http://localhost:8081`

## 🐛 Отладка проблем

### Проблема: "API_URL не задана"

**Решение:**
1. Убедитесь, что файл `.env` существует в папке `mobile/`
2. Проверьте, что в файле есть строка `API_URL=http://localhost:8000`
3. Перезапустите Metro bundler (`npm start`)

### Проблема: "Network Error" или "ECONNREFUSED"

**Решение:**
1. Проверьте, что backend запущен:
   ```bash
   # В другом терминале
   cd backend
   python3 -m uvicorn main:app --reload --host 0.0.0.0 --port 8000
   ```

2. Если тестируете на реальном устройстве, используйте IP-адрес вместо `localhost`:
   ```env
   API_URL=http://192.168.1.100:8000
   ```
   (замените на IP вашей машины)

3. Проверьте, что порт 8000 не заблокирован файрволом

### Проблема: "401 Unauthorized"

**Решение:**
1. Проверьте, что пользователь существует в базе данных
2. Попробуйте зарегистрировать нового пользователя
3. Проверьте, что токен сохраняется (в логах должно быть `🔑 [API] Токен добавлен в заголовки`)

### Проблема: Приложение не запускается

**Решение:**
1. Очистите кэш:
   ```bash
   cd mobile
   npm start -- --clear
   ```

2. Переустановите зависимости:
   ```bash
   rm -rf node_modules
   npm install
   ```

3. Проверьте версию Node.js (должна быть 18+):
   ```bash
   node --version
   ```

## 📊 Проверка логов

### Metro Bundler логи

В терминале, где запущен `npm start`, вы увидите:
- ✅ Успешные запросы: `🌐 [API] Запрос: GET /api/...`
- ✅ Токен добавлен: `🔑 [API] Токен добавлен в заголовки`
- ❌ Ошибки: `❌ [API] Ошибка: ...`

### Логи в приложении

Откройте DevTools в Expo Go:
- **iOS**: Встряхните устройство → "Debug Remote JS"
- **Android**: Нажмите `Cmd+M` (Mac) или `Ctrl+M` (Windows/Linux) → "Debug"
