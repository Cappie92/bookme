# Финальная настройка Maestro тестов

## Проблема и решение

**Проблема:** Maestro запускает Expo Go, но не может автоматически открыть приложение DeDato внутри Expo Go.

**Решение:** Открывать приложение DeDato вручную перед запуском тестов.

## Пошаговая инструкция

### 1. Подготовка

1. Запустите backend (если еще не запущен):
   ```bash
   cd /Users/s.devyatov/DeDato/backend
   # Запустите backend
   ```

2. Запустите Expo в режиме Expo Go:
   ```bash
   cd /Users/s.devyatov/DeDato/mobile
   npx expo start --go
   ```

3. Откройте симулятор (если еще не открыт):
   ```bash
   open -a Simulator
   ```

### 2. Открытие приложения в Expo Go

1. В симуляторе откройте приложение **Expo Go**
2. Найдите проект **"DeDato"** в списке проектов
3. Тапните на **"DeDato"** чтобы открыть приложение
4. Убедитесь, что виден экран входа с текстом "Вход" и "Регистрация"
5. **Убедитесь, что вы вышли из аккаунта** (если были залогинены)

### 3. Запуск тестов

**ВАЖНО:** Приложение DeDato должно быть уже открыто в Expo Go!

```bash
cd /Users/s.devyatov/DeDato/mobile
maestro test .maestro/flows/01-login-success.yaml
```

## Что происходит в тесте

1. `launchApp` - запускает Expo Go (но не открывает DeDato)
2. Тест пытается открыть DeDato тапом на "DeDato" (опционально)
3. Тест вводит данные для входа
4. Тест проверяет успешный вход

## Результаты теста

### Успешный тест:
- ✅ Все шаги выполнены
- ✅ Найден `home-screen` или текст "Привет"

### Неуспешный тест:
- ❌ Не найден `home-screen`
- **Возможные причины:**
  - Приложение не открыто в Expo Go
  - Неправильные учетные данные
  - Backend не запущен
  - Нужно больше времени ожидания

## Альтернативное решение (будущее)

Для полной автоматизации можно:
1. Использовать development build вместо Expo Go
2. Настроить deep linking
3. Использовать координаты для тапа на проект в списке

Но для текущего этапа ручное открытие приложения - это нормально и работает.
