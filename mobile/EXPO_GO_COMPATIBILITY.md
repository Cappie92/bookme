# Проблема с Expo Go: приложение не загружается

## Причина

Expo Go имеет ограничения и не поддерживает:
- Кастомные `bundleIdentifier` и `package`
- Некоторые плагины
- `newArchEnabled: true` может вызывать проблемы

## Решение 1: Исправить конфигурацию для Expo Go

Я отключил `newArchEnabled` в `app.json`. Теперь:

1. **Остановите Expo** (Ctrl+C)

2. **Очистите кеш:**
   ```bash
   cd /Users/s.devyatov/DeDato/mobile
   npx expo start --clear
   ```

3. **Запустите с туннелем:**
   ```bash
   npx expo start --tunnel
   ```

4. **Попробуйте подключиться снова**

## Решение 2: Использовать "Enter URL manually"

1. **В терминале Expo найдите URL** (должен быть показан после запуска)

2. **В Expo Go нажмите "Enter URL manually"**

3. **Вставьте URL** из терминала

4. **Нажмите "Connect"**

## Решение 3: Проверить логи в терминале

При нажатии на сервер в Expo Go, в терминале должны появиться логи:
- Запросы на загрузку bundle
- Ошибки компиляции (если есть)
- Ошибки сети

Если есть ошибки - исправьте их.

## Решение 4: Использовать Development Build (рекомендуется для тестирования)

Expo Go имеет ограничения. Для полноценного тестирования лучше использовать development build:

```bash
cd /Users/s.devyatov/DeDato/mobile
npx expo run:android
# или
npx expo run:ios
```

Это создаст standalone приложение с вашей конфигурацией.

## Решение 5: Для Maestro тестов - использовать эмулятор

Для E2E тестов с Maestro лучше использовать эмулятор/симулятор:

### Android эмулятор:
```bash
npx expo start --tunnel
# Нажмите 'a'
```

### iOS симулятор:
```bash
npx expo start --tunnel
# Нажмите 'i'
```

Эмуляторы работают надежнее и не требуют Expo Go.

## Что проверить

1. **В терминале Expo должны быть:**
   - ✅ QR-код
   - ✅ URL (exp://...)
   - ✅ Нет ошибок компиляции

2. **При нажатии на сервер в Expo Go:**
   - В терминале должны появиться запросы
   - Если ошибки - исправьте их

3. **Попробуйте "Enter URL manually"** - это часто работает лучше чем QR-код
