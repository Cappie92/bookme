# 🚨 Быстрое решение Network Error

## ✅ Backend доступен (проверено)

Backend отвечает на `http://192.168.0.194:8001/health` ✅

---

## 🔍 Что проверить сейчас

### 1. Посмотри логи в Metro bundler

Открой консоль Metro bundler (где запущен `npm start`) и попробуй войти в приложение.

**Найди в логах строку:**
```
Network Error: {
  message: "...",
  url: "/api/auth/login",
  baseURL: "http://192.168.0.194:8001",
  fullURL: "http://192.168.0.194:8001/api/auth/login",
  code: "..."
}
```

**Скопируй сюда код ошибки (code) и сообщение (message)**

---

### 2. Проверь API_URL в консоли

При запуске Metro bundler должна быть строка:
```
✅ API_URL настроен: http://192.168.0.194:8001
```

**Если её нет или URL неправильный:**
- Проверь `mobile/.env` файл
- Перезапусти Metro bundler

---

### 3. Проверь с телефона напрямую

Открой в браузере на телефоне:
```
http://192.168.0.194:8001/health
```

**Если не открывается:**
- Телефон и компьютер в разных сетях
- Firewall блокирует порт 8001
- Неправильный IP адрес

---

### 4. Попробуй tunnel режим

Если ничего не помогает:

```bash
cd mobile
npm start -- --tunnel
```

Это создаст публичный URL через Expo сервис.

## 🔧 Возможные решения

### Если код ошибки `ECONNREFUSED`:
- Backend не запущен или недоступен
- Проверь: `curl http://192.168.0.194:8001/health`

### Если код ошибки `ENOTFOUND`:
- Неправильный IP адрес в `.env`
- Проверь IP: `ipconfig getifaddr en0` (macOS)

### Если код ошибки `ETIMEDOUT`:
- Firewall блокирует порт
- Телефон и компьютер в разных сетях
- Решение: используй tunnel режим

### Если ошибка CORS:
- Backend не в development режиме
- Проверь: `echo $ENVIRONMENT` (должно быть `development`)
