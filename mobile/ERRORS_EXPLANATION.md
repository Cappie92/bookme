# Объяснение ошибок при запуске

## ✅ Нормальные ошибки (можно игнорировать)

### 1. Ошибки 401 (Not authenticated)
```
ERROR  API Error: {"message": "Not authenticated", "status": 401}
```

**Причина:** Пользователь не залогинен, поэтому все защищенные API endpoints возвращают 401.

**Решение:** Это нормально! Просто залогиньтесь через экран входа.

**Что исправлено:**
- Добавлена проверка авторизации перед загрузкой данных на главном экране
- Теперь данные не загружаются, если пользователь не авторизован

### 2. Ошибка 404 на `/api/bookings`
```
ERROR  API Error: {"message": "Not Found", "status": 404, "url": "/api/bookings"}
```

**Причина:** Для клиентов используется другой endpoint (`/api/client/bookings/`), но когда пользователь не залогинен, роль undefined, и используется неправильный endpoint.

**Решение:** Исправлено - теперь данные не загружаются, если пользователь не авторизован.

## ⚠️ Предупреждения (не критично)

### 1. New Architecture Warning
```
WARN  🚨 React Native's New Architecture is always enabled in Expo Go
```

**Причина:** В `app.json` было `newArchEnabled: false`, но Expo Go всегда использует New Architecture.

**Решение:** Удалено `newArchEnabled: false` из `app.json`. Это предупреждение можно игнорировать - оно не влияет на работу приложения.

### 2. Роутинг "notes"
```
WARN  [Layout children]: No route named "notes" exists
```

**Причина:** В `_layout.tsx` была попытка явно указать роут `notes`, но Expo Router автоматически обрабатывает `app/notes/index.tsx` как `/notes`.

**Решение:** Убрано явное указание роута - Expo Router автоматически обработает его.

## 📝 Что нужно сделать

1. **Залогиньтесь** через экран входа (`/login`)
2. После входа все ошибки 401 и 404 должны исчезнуть
3. Данные начнут загружаться автоматически

## 🔍 Проверка

После логина проверьте:
- ✅ Нет ошибок 401
- ✅ Нет ошибок 404
- ✅ Данные загружаются (бронирования, избранное)
- ✅ Роутинг работает (`/notes`, `/settings`)
