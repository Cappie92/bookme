# Maestro E2E Тесты для DeDato Mobile

## ⚠️ ВАЖНО: Перед запуском тестов

**Обязательно откройте приложение DeDato в Expo Go вручную!**

1. Запустите Expo: `npx expo start --go`
2. В симуляторе откройте Expo Go
3. Найдите проект "DeDato" в списке проектов
4. Тапните на "DeDato" чтобы открыть приложение
5. Убедитесь, что виден экран входа
6. **Только после этого** запускайте тесты

## Почему это необходимо?

Maestro может запустить Expo Go через `launchApp`, но не может автоматически открыть конкретное приложение внутри Expo Go. Поэтому нужно открыть приложение вручную перед запуском тестов.

## Запуск тестов

### Все тесты:
```bash
cd /Users/s.devyatov/DeDato/mobile
npm run test:e2e
```

### Один тест:
```bash
maestro test .maestro/flows/01-login-success.yaml
```

## Доступные тесты

1. **01-login-success.yaml** - Тест успешного входа
2. **02-login-error.yaml** - Тест ошибки входа
3. **03-navigate-to-bookings.yaml** - Тест перехода в бронирования
4. **04-open-booking-details.yaml** - Тест открытия деталей бронирования
5. **05-logout.yaml** - Тест выхода из аккаунта

## Решение проблем

### Проблема: "Element not found"

**Причина:** Приложение не открыто в Expo Go.

**Решение:** Откройте приложение DeDato в Expo Go вручную перед запуском тестов.

### Проблема: "Assertion is false: id: home-screen is visible"

**Причина:** Вход не прошел успешно или нужно больше времени ожидания.

**Решение:**
1. Проверьте учетные данные в тесте
2. Убедитесь, что backend запущен
3. Проверьте логи в консоли Expo

## Альтернативное решение (в будущем)

Для автоматизации можно:
1. Использовать development build вместо Expo Go
2. Настроить deep linking для автоматического открытия приложения
3. Использовать координаты для тапа на проект в списке Expo Go
