# Исправление проблемы подключения Expo Go

## Проблема: Сервер виден, но приложение не загружается

Если сервер виден в Expo Go, но при нажатии ничего не происходит, попробуйте следующие решения:

## Решение 1: Использовать "Enter URL manually"

1. **В Expo Go нажмите "Enter URL manually"**

2. **В терминале Expo найдите URL** (должен быть что-то вроде):
   ```
   exp://u.expo.dev/...
   ```
   или
   ```
   exp://192.168.x.x:8081
   ```

3. **Скопируйте полный URL** из терминала

4. **Вставьте в Expo Go** и нажмите "Connect"

## Решение 2: Проверить логи Expo

В терминале где запущен `npx expo start --tunnel` должны быть логи подключения.

Если при нажатии на сервер появляются ошибки - проверьте:
- Ошибки сети
- Ошибки загрузки bundle
- Ошибки конфигурации

## Решение 3: Очистить кеш Expo Go

1. **В Expo Go:** Settings → Clear cache
2. **Или переустановите Expo Go**

## Решение 4: Проверить что приложение запущено правильно

Убедитесь что в терминале Expo нет ошибок:
- Должен быть показан QR-код
- Должен быть показан URL
- Не должно быть ошибок компиляции

## Решение 5: Использовать эмулятор/симулятор (для тестирования)

Для Maestro тестов лучше использовать эмулятор:

### Android:
```bash
npx expo start --tunnel
# Нажмите 'a'
```

### iOS:
```bash
npx expo start --tunnel
# Нажмите 'i'
```

Эмуляторы работают надежнее для тестирования.

## Решение 6: Проверить scheme в app.json

Убедитесь что в `app.json` указан правильный scheme:
```json
{
  "expo": {
    "scheme": "dedato"
  }
}
```

## Что проверить в терминале Expo

При запуске `npx expo start --tunnel` должны быть:
- ✅ QR-код отображается
- ✅ URL показан (exp://...)
- ✅ Нет ошибок компиляции
- ✅ Metro bundler запущен

Если есть ошибки - исправьте их перед подключением.
