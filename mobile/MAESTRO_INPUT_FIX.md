# Исправление проблемы с вводом данных в Maestro

## Проблема

При вводе данных в поля формы:
- Телефон вводится неправильно (последняя цифра не помещается)
- Пароль вводится не полностью

## Причина

1. Поля могут содержать предзаполненный текст (например, "+7")
2. При вводе нового текста он добавляется к существующему, а не заменяет его
3. Нужно сначала очистить поле перед вводом

## Решение

### Вариант 1: Использование longPress для выделения

```yaml
- tapOn:
    id: "phone-input"
- longPressOn:
    id: "phone-input"
- tapOn:
    text: "Select All"
    optional: true
- inputText: "+79991234567"
```

### Вариант 2: Удаление символов по одному

Если Select All не работает, можно удалить символы по одному:

```yaml
- tapOn:
    id: "phone-input"
# Удаляем предзаполненный "+7" (2 символа)
- pressKey: [Delete]
- pressKey: [Delete]
- inputText: "9991234567"
```

### Вариант 3: Использование eraseText (если поддерживается)

```yaml
- tapOn:
    id: "phone-input"
- eraseText
- inputText: "+79991234567"
```

## Текущее решение в тесте

Используется комбинация:
1. Тап на поле
2. Длинное нажатие для вызова контекстного меню
3. Выбор "Select All" (если доступно)
4. Ввод нового текста (заменит выделенный)

## Проверка

После ввода данных проверьте в симуляторе:
- ✅ Телефон введен полностью: `+79991234567`
- ✅ Пароль введен полностью: `password123` (11 символов, видно как 11 точек)

## Альтернатива: Изменить логику ввода в приложении

Можно изменить компонент `TextInput` чтобы он автоматически очищался при фокусе:

```typescript
<TextInput
  testID="phone-input"
  onFocus={() => {
    // Очистить поле при фокусе (опционально)
  }}
  // ...
/>
```

Но это может ухудшить UX для пользователей, поэтому лучше исправить тест.
