# Запуск Maestro E2E тестов

## Предварительные требования

1. ✅ iOS симулятор запущен
2. ✅ Приложение открыто в Expo Go в симуляторе
3. ✅ Backend запущен и доступен по `http://localhost:8001`

## Важно перед запуском тестов

**Убедитесь, что вы вышли из аккаунта!** Если вы залогинены, тесты не смогут найти экран входа.

### Как выйти из аккаунта:
1. Откройте приложение в симуляторе
2. Нажмите на "Настройки" в нижней навигации
3. Нажмите "Выйти"
4. Подтвердите выход

## Запуск тестов

### Запуск всех тестов:
```bash
cd /Users/s.devyatov/DeDato/mobile
npm run test:e2e
```

### Запуск конкретного теста:
```bash
cd /Users/s.devyatov/DeDato/mobile
maestro test .maestro/flows/01-login-success.yaml
```

### Запуск тестов для iOS:
```bash
cd /Users/s.devyatov/DeDato/mobile
npm run test:e2e:ios
```

## Доступные тесты

1. **01-login-success.yaml** - Тест успешного входа
2. **02-login-error.yaml** - Тест ошибки входа
3. **03-navigate-to-bookings.yaml** - Тест перехода в бронирования
4. **04-open-booking-details.yaml** - Тест открытия деталей бронирования
5. **05-logout.yaml** - Тест выхода из аккаунта

## Решение проблем

### Проблема: "Assertion is false: id: login-screen-title is visible"

**Причина:** Пользователь уже залогинен, или приложение не открыто в Expo Go.

**Решение:**
1. Убедитесь, что приложение открыто в симуляторе
2. Выйдите из аккаунта вручную
3. Перезапустите тесты

### Проблема: "No devices connected"

**Причина:** Симулятор не запущен.

**Решение:**
```bash
open -a Simulator
# Или через Xcode: Xcode → Open Developer Tool → Simulator
```

### Проблема: "Failed to launch app"

**Причина:** Приложение не установлено в Expo Go.

**Решение:**
1. Убедитесь, что Expo Go установлен в симуляторе
2. Запустите приложение вручную через Expo Go
3. Перезапустите тесты

## Отладка

Если тесты не проходят, проверьте:
1. Скриншоты в `/Users/s.devyatov/.maestro/tests/`
2. Логи в консоли
3. Убедитесь, что все testID правильно установлены в компонентах
