# Отладка Metro Bundler

## Проблема: Metro ждет подключения, но запросов нет

Если Metro показывает "waiting on exp://..." но запросов нет, это может означать:

1. **Приложение не может подключиться к туннелю**
2. **Ошибки компиляции блокируют загрузку**
3. **Проблемы с сетью/туннелем**

## Решение 1: Проверить что bundle доступен

Попробуйте открыть в браузере:
```
http://mzzfsqi-p1lula-8081.exp.direct/index.bundle?platform=android
```

Если открывается - bundle доступен.
Если ошибка - проблема в компиляции.

## Решение 2: Запустить Metro с подробными логами

Остановите текущий Metro и запустите с verbose логами:

```bash
cd /Users/s.devyatov/DeDato/mobile
npx expo start --tunnel --clear
```

Затем при попытке подключения смотрите:
- Появляются ли запросы в терминале
- Есть ли ошибки компиляции
- Есть ли ошибки сети

## Решение 3: Проверить логи компиляции

Metro должен компилировать TypeScript/JavaScript. Если есть ошибки - они будут показаны.

Проверьте:
- Нет ли ошибок TypeScript
- Нет ли ошибок импортов
- Нет ли ошибок в зависимостях

## Решение 4: Использовать эмулятор (рекомендуется)

Для тестирования лучше использовать эмулятор:

### Android эмулятор:
```bash
npx expo start --tunnel --clear
# Нажмите 'a'
```

Эмулятор:
- ✅ Не требует Expo Go
- ✅ Работает напрямую с Metro
- ✅ Всегда доступен для Maestro
- ✅ Не зависит от туннеля

### iOS симулятор:
```bash
npx expo start --tunnel --clear
# Нажмите 'i'
```

## Решение 5: Проверить что приложение компилируется

Попробуйте открыть bundle напрямую в браузере:
```
http://mzzfsqi-p1lula-8081.exp.direct/index.bundle?platform=android&dev=true
```

Если видите JavaScript код - компиляция работает.
Если ошибка - проблема в коде или конфигурации.

## Решение 6: Проверить консоль в Expo Go

В Expo Go может быть встроенная консоль:
1. Откройте Expo Go
2. Найдите настройки/консоль
3. Проверьте ошибки там

## Что проверить

1. **В терминале Metro:**
   - Должны быть логи компиляции
   - При подключении должны появиться запросы
   - Не должно быть ошибок

2. **В браузере:**
   - Попробуйте открыть URL bundle
   - Проверьте что он загружается

3. **В Expo Go:**
   - Проверьте консоль/логи
   - Проверьте настройки сети

## Рекомендация

Для Maestro тестов **лучше использовать эмулятор** - он работает надежнее и не требует Expo Go.
