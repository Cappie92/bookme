# Чеклист перед первой сборкой

## ✅ 1. Обновление идентификаторов

### iOS Bundle Identifier

Открой `mobile/app.config.ts` и замени:

```typescript
ios: {
  bundleIdentifier: 'com.твоякомпания.dedato', // Замени на реальный
  // ...
}
```

**Рекомендации:**
- Используй обратный домен: `com.твоякомпания.dedato`
- Только латиница, без пробелов
- Примеры: `com.dedato.app`, `com.mycompany.dedato`, `ru.dedato.app`

### Android Package Name

В том же файле замени:

```typescript
android: {
  package: 'com.твоякомпания.dedato', // Замени на реальный
  // ...
}
```

**Важно:** iOS и Android идентификаторы должны совпадать (или быть максимально похожими).

## ✅ 2. Проверка иконок и splash

Проверь файлы в `mobile/assets/`:

- ✅ `icon.png` — должна быть 1024×1024 px
- ✅ `adaptive-icon.png` — должна быть 1024×1024 px  
- ✅ `splash-icon.png` — рекомендуется 2048×2048 px или больше
- ✅ `favicon.png` — для web (48×48 px)

**Для первой dev-сборки:** Если иконки временные/заглушки — это нормально, можно оставить. Для production-сборок обязательно замени на финальные.

**Проверка размеров:**
```bash
cd mobile
# macOS
file assets/icon.png
# или через ImageMagick (если установлен)
identify assets/icon.png
```

## ✅ 3. Проверка API_URL

Открой `mobile/.env` и проверь:

### Для локального тестирования (dev-сборка):

```env
API_URL=http://<IP_ТВОЕЙ_МАШИНЫ>:8000
```

**Как узнать IP:**
- macOS/Linux: `ifconfig | grep "inet "` или `ipconfig getifaddr en0`
- Windows: `ipconfig`
- Ищи IP в локальной сети (обычно начинается с 192.168.x.x или 10.x.x.x)

**Важно:** 
- Убедись, что backend запущен и доступен по этому IP
- Убедись, что телефон и компьютер в одной Wi-Fi сети
- Для production позже замени на публичный домен: `https://api.dedato.com`

## 🚀 4. Первая dev-сборка через EAS

### Шаг 1: Авторизация в Expo

```bash
cd mobile
npx eas login
```

Войди в существующий аккаунт или создай новый.

### Шаг 2: Инициализация EAS (если ещё не делал)

```bash
npx eas init
```

Это создаст проект в Expo Dashboard (если ещё не создан).

### Шаг 3: Dev-сборка для Android

```bash
npx eas build --platform android --profile development
```

**Что произойдёт:**
- EAS соберёт APK с development client
- После завершения получишь ссылку на скачивание
- Установи APK на Android-устройство и открой приложение

### Шаг 4: Dev-сборка для iOS (если есть macOS + Apple Developer аккаунт)

```bash
npx eas build --platform ios --profile development
```

**Требования:**
- macOS
- Apple Developer аккаунт (бесплатный для разработки)
- Xcode (для локальной сборки) или EAS Build (облачная сборка)

**После сборки:**
- iOS: получишь ссылку для установки через Expo Go или TestFlight
- Android: получишь APK для прямой установки

## 🧪 5. Чеклист тестирования на устройстве

### ✅ Авторизация

- [ ] Запуск приложения → автоматический редирект на `/login` (если не залогинен)
- [ ] Ввод телефона и пароля → успешный вход
- [ ] После входа → переход на главный экран (`/`)
- [ ] Закрытие и повторный запуск приложения → автоматический вход через сохранённый токен (без ввода пароля)

### ✅ Главный экран

- [ ] Отображается приветствие с именем пользователя (или "Привет!")
- [ ] Если есть предстоящие бронирования → показывается карточка "Ближайшая запись"
- [ ] Если нет бронирований → показывается "Нет предстоящих бронирований"
- [ ] Если есть подписка → показывается статус подписки с бейджем
- [ ] Если нет подписки → показывается "Подписка не найдена"
- [ ] Кнопки "Быстрые действия" работают и ведут на соответствующие экраны

### ✅ Бронирования

- [ ] Переход в `/bookings` → загружается список бронирований
- [ ] Pull-to-refresh → обновление списка
- [ ] Тап по карточке бронирования → открывается детальный экран `/bookings/[id]`
- [ ] На детальном экране отображаются все данные бронирования
- [ ] Кнопка "Назад" → возврат к списку

### ✅ Подписки

- [ ] Переход в `/subscriptions` → загружается информация о подписке
- [ ] Отображается название плана, статус, дата окончания
- [ ] Отображаются лимиты (филиалы, сотрудники, бронирования)
- [ ] Pull-to-refresh → обновление данных
- [ ] Если подписки нет → показывается соответствующее сообщение

### ✅ Настройки

- [ ] Переход в `/settings` → отображается профиль пользователя
- [ ] Аватар с инициалами отображается корректно
- [ ] Отображаются имя, роль, телефон, email
- [ ] Кнопка "Выйти из аккаунта" → показывается Alert с подтверждением
- [ ] После подтверждения → выход, очистка токена, редирект на `/login`

### ✅ Общее

- [ ] Навигация между экранами работает плавно
- [ ] Нет критических ошибок в консоли
- [ ] API-запросы выполняются успешно (проверь Network tab в DevTools, если доступен)
- [ ] Приложение не крашится при переключении между экранами

## 📝 Заметки по тестированию

### Если что-то не работает:

1. **API не отвечает:**
   - Проверь, что backend запущен
   - Проверь `API_URL` в `.env`
   - Проверь, что телефон и компьютер в одной сети
   - Проверь firewall настройки

2. **Авторизация не работает:**
   - Проверь, что токен сохраняется (можно проверить через AsyncStorage/SecureStore)
   - Проверь, что токен добавляется в заголовки запросов
   - Проверь логи в консоли Metro bundler

3. **Навигация не работает:**
   - Проверь, что Expo Router правильно настроен
   - Проверь, что все экраны существуют в `app/`

4. **Стили выглядят странно:**
   - Проверь, что все компоненты UI-кита импортированы правильно
   - Проверь, что SafeAreaView работает корректно на устройстве
