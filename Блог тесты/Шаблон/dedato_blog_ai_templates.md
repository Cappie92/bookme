# DeDato Blog AI Templates

Этот файл объединяет два рабочих шаблона для генерации статьи в блог DeDato:

1. **Полный шаблон** — для более детальной и ручной подготовки статьи.
2. **Упрощённый шаблон** — для быстрого AI-generation workflow.

Оба шаблона рассчитаны на импорт статьи в блог в формате **Markdown + YAML frontmatter**.

---

## Как использовать

- Генерируйте статью через ИИ **строго по одному из шаблонов ниже**.
- На выходе должен получиться **один `.md` файл**.
- Этот файл загружается в импорт статьи в админке.
- Система парсит frontmatter, раскладывает поля по форме и подставляет HTML-контент в редактор.

---

## Общие правила для ИИ

1. Не добавлять поля, которых нет в шаблоне.
2. `excerpt` и `meta_description` делать не длиннее 160 символов.
3. Если `slug` неочевиден — оставлять пустым.
4. `status` оставлять `draft`.
5. Писать основной текст в чистом Markdown.
6. Не вставлять опасный HTML и скрипты.
7. Если поле неизвестно — оставлять пустым, а не выдумывать структуру.

# Шаблон 1. Полный

```markdown
---
title: ""
slug: ""
subtitle: ""
excerpt: ""

meta_title: ""
meta_description: ""
canonical_url: ""
robots_noindex: false
robots_nofollow: false

og_title: ""
og_description: ""
og_image: ""

twitter_title: ""
twitter_description: ""
twitter_image: ""

cover_image: ""
cover_image_alt: ""

tags:
  - ""

status: draft
published_at: ""
scheduled_at: ""
---

# Введение

Короткое вступление в 1–2 абзаца.

## Подзаголовок 1

Основной текст раздела.

## Подзаголовок 2

- Первый пункт
- Второй пункт
- Третий пункт

## Подзаголовок 3

Текст раздела с примерами, пояснениями и выводами.

## Итоги

Краткий вывод статьи и практическое резюме.
```

### Когда использовать
- когда нужна полноценная SEO-подготовка;
- когда статья важная и будет редактироваться вручную;
- когда заранее известны OG/Twitter/канонические поля.

# Шаблон 2. Упрощённый

```markdown
---
title: "Заголовок статьи"
slug: ""
subtitle: "Необязательно"
excerpt: "Короткий анонс статьи до 160 символов."

meta_title: "SEO-заголовок до ~60 символов"
meta_description: "SEO-описание до 160 символов."

cover_image: ""
cover_image_alt: ""

tags:
  - "тег-1"
  - "тег-2"

status: draft
---

# Введение

2–3 абзаца, которые быстро вводят в тему статьи.

## Основная мысль 1

Подробный текст.

## Основная мысль 2

Подробный текст.

## Практические выводы

Список конкретных рекомендаций:

- пункт 1
- пункт 2
- пункт 3

## Итог

Короткий финальный вывод.
```

### Когда использовать
- для повседневной генерации через ИИ;
- когда важна скорость;
- когда SEO-поля нужны только в базовом виде;
- когда не хочется перегружать генератор.

---

## Какие поля реально важны

### Обязательные
- `title`
- основной текст статьи

### Очень желательные
- `excerpt`
- `meta_title`
- `meta_description`
- `tags`

### Можно оставить пустыми
- `slug`
- `subtitle`
- `canonical_url`
- `og_*`
- `twitter_*`
- `cover_image`
- `cover_image_alt`
- `published_at`
- `scheduled_at`

---

## Практическая рекомендация

Для работы лучше использовать такой принцип:

- **80% статей** — по упрощённому шаблону
- **20% ключевых статей** — по полному шаблону

Так ты сохраняешь скорость, но не теряешь контроль над важными публикациями.
