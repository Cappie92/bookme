## 📋 Что делать дальше (по порядку)

### Шаг 1: Применить миграции базы данных

Миграции создадут таблицу `payments` и конвертируют балансы с копеек на рубли.

**Выполните команду:**

```bash
cd /Users/s.devyatov/DeDato/backend
python3 -m alembic upgrade head
```

**Что произойдет:**
1. Создастся таблица `payments` для хранения платежей
2. Конвертируются все балансы с копеек на рубли:
   - `user_balances.balance`
   - `balance_transactions.amount`, `balance_before`, `balance_after`
   - `subscription_reservations.reserved_amount` (было `reserved_kopecks`)

**⚠️ Важно:**
- Перед миграцией сделайте резервную копию базы данных
- Миграция необратима (можно откатить, но лучше иметь бэкап)

**Если возникнут ошибки:**
- Проверьте, что база данных не заблокирована
- Убедитесь, что нет активных подключений к БД
- Проверьте логи ошибок

---

### Шаг 2: Проверить доступность Result URL

Поскольку вы используете реальный домен `dedato.ru`, нужно убедиться, что:

1. **Сервер запущен и доступен:**
   ```bash
   # Проверьте, что сервер работает
   curl https://dedato.ru/api/payments/robokassa/result
   ```

2. **Эндпоинт доступен из интернета:**
   - Robokassa должна иметь возможность отправлять POST запросы на ваш Result URL
   - Убедитесь, что нет файрвола, блокирующего запросы

3. **HTTPS настроен:**
   - Robokassa требует HTTPS для боевого режима
   - Убедитесь, что SSL сертификат валиден

---

### Шаг 3: Проверить настройки в .env

Убедитесь, что в файле `backend/.env` указаны правильные URL:

```env
# Для боевого режима (если используете dedato.ru)
ROBOKASSA_RESULT_URL=https://dedato.ru/api/payments/robokassa/result
ROBOKASSA_SUCCESS_URL=https://dedato.ru/payment/success
ROBOKASSA_FAIL_URL=https://dedato.ru/payment/failed
```

**Или если у вас раздельные домены:**
```env
ROBOKASSA_RESULT_URL=https://api.dedato.ru/api/payments/robokassa/result
ROBOKASSA_SUCCESS_URL=https://app.dedato.ru/payment/success
ROBOKASSA_FAIL_URL=https://app.dedato.ru/payment/failed
```

### Шаг 4: Найти MerchantLogin

В файле `.env` нужно заменить `your_merchant_login_here` на реальный логин.

**Где найти:**
1. Войдите в личный кабинет Robokassa
2. Перейдите в "Настройки" → "Основные настройки"
3. Найдите "Логин магазина" или "MerchantLogin"
4. Скопируйте его в `.env`

**Или:**
- Обычно это ваш email или специальный логин, который вы указали при регистрации

### Шаг 5: Тестирование

#### 5.1. Тестовый платеж

1. **Установите тестовый режим:**
   ```env
   ROBOKASSA_IS_TEST=true
   ```

2. **Войдите в личный кабинет мастера:**
   - Перейдите в "Мой тариф"
   - Выберите план подписки
   - Нажмите "Перейти к оплате"

3. **Используйте тестовую карту:**
   - Номер: `4111111111111111`
   - Любая дата (будущая)
   - Любой CVV

4. **Проверьте результат:**
   - Должна открыться страница успеха
   - Платеж должен появиться в базе данных
   - Подписка должна быть создана (но не активирована)

#### 5.2. Проверка базы данных

```sql
-- Проверьте созданные платежи
SELECT * FROM payments ORDER BY created_at DESC LIMIT 5;

-- Проверьте подписки
SELECT id, user_id, status, is_active, plan_id FROM subscriptions ORDER BY created_at DESC LIMIT 5;

-- Проверьте балансы (должны быть в рублях, не копейках)
SELECT id, user_id, balance FROM user_balances LIMIT 5;
```

### Шаг 6: Переход в боевой режим

Когда все протестировано:

1. **Измените в `.env`:**
   ```env
   ROBOKASSA_IS_TEST=false
   ```

2. **Перезапустите сервер**

3. **Проверьте настройки в Robokassa:**
   - Убедитесь, что Result URL указывает на боевой домен
   - Проверьте, что метод = POST

## 🔍 Проверка работоспособности

### Проверка 1: Эндпоинт доступен

```bash
curl -X POST https://dedato.ru/api/payments/robokassa/result \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "OutSum=100&InvId=123&SignatureValue=test"
```

Должен вернуть ошибку о неверной подписи (это нормально, значит эндпоинт работает).

### Проверка 2: Логи сервера

После тестового платежа проверьте логи сервера:
- Должны быть запросы от Robokassa на Result URL
- Должны быть записи о создании платежей

### Проверка 3: База данных

Проверьте, что:
- Таблица `payments` создана
- Балансы конвертированы в рубли
- Новые платежи создаются корректно

---

## ⚠️ Важные замечания

1. **Result URL должен быть POST:**
   - Проверьте в настройках Robokassa
   - Это критически важно!

2. **HTTPS обязателен для боевого режима:**
   - Robokassa требует HTTPS
   - Убедитесь, что SSL сертификат валиден

3. **Доступность из интернета:**
   - Result URL должен быть доступен из интернета
   - Проверьте файрвол и настройки сервера

4. **Резервная копия:**
   - Перед миграцией сделайте бэкап базы данных
   - Миграция конвертирует все балансы
