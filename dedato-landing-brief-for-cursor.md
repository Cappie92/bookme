## Структура секций (сверху вниз)

1. **Nav** — лого `dedato` (SVG иконка + текст), пункты: Возможности / Как это работает / Приложения / Тарифы / Блог. Справа: `Войти` (ghost) + `Попробовать бесплатно` (primary).
2. **Hero** — 2 колонки (лево: текст + CTA + соц-пруф с аватарами; право: мокап расписания с двумя плавающими мини-карточками «Выручка» и «Новая запись»).
   - H1: «Онлайн-запись и клиенты — в одном **сервисе**» (слово «сервисе» выделено зелёным с подчёркиванием-хайлайтом).
   - Sub: «Расписание, онлайн-бронирование, клиентская база, скидки и финансы. Работает в браузере, на iPhone и Android.»
   - CTA: `Попробовать бесплатно` → открыть `openAuthModal('master', 'register')`; `Демо-кабинет` → `navigate('/demo/master')`.
   - Бейдж сверху h1: «Для частных мастеров услуг» с пульсирующей зелёной точкой.
3. **Features** (6 карточек, 3×2) — в существующем файле уже есть массив `FEATURES`, оставь его, но переиспользуй новые иконки и стили карточек (hover: lift + тень + зелёный бордер). Иконки — Heroicons outline.
4. **How it works** (3 шага) — на светлом фоне `#F4F1EF`. Круглый номер шага с зелёной обводкой-ореолом (`box-shadow: 0 0 0 6px #DFF5EC`).
5. **Devices** (4 карточки) — `Web` — активная, градиент `green-500 → green-600`, белая иконка и текст. Остальные (App Store, Google Play, RuStore) — `Скоро`.
6. **Available / Soon** (2 колонки) — слева green-50 фон, чеклист с зелёными галочками в кружках; справа серый фон, часики.
7. **Final CTA** — тёмный `#1B1B1B` с зеленоватым радиальным свечением в углах, скруглённый блок 28px, белый заголовок и две кнопки.
8. **Footer** — лого слева, копирайт справа.

## Новое по сравнению с текущим `Home.jsx`

Текущий `Home.jsx` — ок, но с «placeholder» местом под product shot. В макете этот placeholder заменён на **реальный мокап расписания**, собранный из HTML/CSS (без картинок): шапка «Моё расписание», полоска дней недели с выделенным текущим днём, 3–4 слота (booked / booked-2 / free) с временем, именем услуги и клиентом. Рядом — 2 плавающие карточки с «Выручка за апрель +92 400 ₽» и «Новая запись Ольга П. · маникюр».

Это большой визуальный апгрейд — именно его нужно портировать в React. Можно вынести мокап в отдельный компонент `HeroScheduleMock.jsx`.

## Acceptance checklist

- [ ] Верстка совпадает с `dedato-landing-preview.html` на 1440px (desktop) и 390px (mobile iPhone 14).
- [ ] Все цвета — через токены, не хардкод (`#4CAF50` только в свг-иконках логотипа допустимо).
- [ ] Шрифт `Jost` загружен; заголовки `font-weight: 700`, basic текст `400–500`.
- [ ] CTA-кнопки имеют «double-shadow» эффект: `box-shadow: 0 2px 0 var(--green-700), 0 4px 14px -2px rgba(76,175,80,.4)`.
- [ ] Hero-мокап расписания — это настоящий HTML/CSS, а не картинка.
- [ ] Плавающие карточки в hero имеют `transform: rotate(±2–3deg)`.
- [ ] «Слово «сервисе»» (в h1) подчёркнуто зелёным хайлайтом-бликом, а не обычным underline.
- [ ] Mobile < 600px: все сетки сжимаются в одну колонку.
- [ ] Семантика: `<nav>`, `<main>`, `<section>`, `<h1>/<h2>/<h3>`, `aria-label` у иконочных кнопок.
- [ ] a11y: контраст текста ≥ 4.5, фокус-кольца для кнопок и ссылок.
